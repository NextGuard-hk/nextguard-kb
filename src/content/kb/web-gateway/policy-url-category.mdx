## Overview

URL Category Policy allows you to control web access based on the category of websites users try to visit. The Nextguard WSG maintains a database of 100+ URL categories that is continuously updated.

---

## URL Category Database

The WSG categorizes websites into groups including:

**Security Threats**
- Malware / Phishing / Botnet Command & Control
- Newly Registered Domains
- Suspicious Content

**Productivity**
- Social Media (Facebook, Twitter, Instagram, TikTok)
- Online Gaming
- Streaming Media (YouTube, Netflix)
- Job Search

**Legal / Compliance**
- Adult Content
- Gambling
- Weapons / Violence
- Illegal Activities

**Business**
- Finance & Banking
- Healthcare
- Government
- News & Media

**Uncategorized:** Sites not yet in the database are handled by configurable default action.

## Creating a URL Category Policy

1. Navigate to **Policy > Web Access Policy**
2. Click **Add Policy**
3. Configure:

| Field | Description |
|-------|-------------|
| **Name** | Descriptive policy name |
| **Priority** | Lower number = higher priority |
| **Users/Groups** | AD groups or IP ranges to apply to |
| **Schedule** | Time-based enforcement (e.g., work hours only) |
| **Categories** | Select URL categories to act on |
| **Action** | Allow / Block / Monitor / Warn |
| **Redirect URL** | Custom block page URL (optional) |

4. Click **Save & Apply**

---

## Policy Actions

| Action | Behavior |
|--------|----------|
| **Allow** | Permit access, optionally log |
| **Block** | Deny access, show block page |
| **Monitor** | Permit access and log (no blocking) |
| **Warn** | Show warning page; user can continue |
| **Quota** | Allow with time/data limit per day |

## Custom URL Lists

You can override the category database with custom lists:

**Allowlist (Whitelist)**
1. Navigate to **Policy > Custom URL Lists > Allowlist**
2. Add domains or URLs that should always be allowed
3. Allowlist overrides category blocking

**Blocklist (Blacklist)**
1. Navigate to **Policy > Custom URL Lists > Blocklist**
2. Add domains or URLs to always block regardless of category

**Custom Category**
1. Navigate to **Policy > Custom URL Lists > Custom Category**
2. Create named groups of URLs (e.g., "Approved Cloud Apps")
3. Use in policy rules like any built-in category

---

## Time-Based Policies

Create different policies for work hours vs. off-hours:

1. Navigate to **Policy > Schedules**
2. Create a schedule (e.g., Mon-Fri 9am-6pm)
3. In the policy rule, set **Schedule** to your created schedule
4. Create a second rule (lower priority) for outside work hours

---

## Safe Search Enforcement

To enforce Safe Search on search engines:

1. Navigate to **Policy > Safe Search**
2. Enable **Enforce Safe Search**
3. The WSG will rewrite search requests to include safe search parameters for Google, Bing, YouTube, etc.

---

## Block Page Customization

1. Navigate to **System > Block Page**
2. Customize the HTML/CSS of the block page
3. Include your company logo and helpdesk contact
4. Users who visit blocked sites will see this page

---

## Reporting on URL Access

1. Navigate to **Reports > Web Browsing**
2. View:
   - Top categories accessed
   - Top blocked categories
   - User activity by category
   - Bandwidth usage by category
3. Export reports as PDF or CSV
