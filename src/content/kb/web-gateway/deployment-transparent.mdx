## Overview

In Transparent Proxy mode, HTTP/HTTPS traffic is redirected to the WSG using firewall or switch policy (typically via WCCP or Policy-Based Routing). Clients do not need to be configured with proxy settings.

**Recommended for:** Environments where you cannot modify client settings but want proxy-style inspection.

---

## Comparison with Inline Mode

| Feature | Inline | Transparent Proxy |
|---------|--------|-------------------|
| Client configuration | None | None |
| Traffic redirect method | Physical placement | WCCP / PBR |
| Failover behavior | Fail-open (bypass) | Controlled by router |
| Performance | Higher | Slightly lower |

---

## Prerequisites

- [ ] WSG deployed and reachable from the router/firewall
- [ ] Router/firewall supports WCCP or Policy-Based Routing (PBR)
- [ ] WSG traffic interface in the same subnet or routable from gateway
- [ ] Ports 80 and 443 redirected to WSG proxy port (8080)

---

## Step 1: Enable Transparent Proxy Mode

1. Log into WSG management console
2. Navigate to **Network > Deployment Mode**
3. Select **Transparent Proxy**
4. Enable WCCP (if using Cisco) or configure PBR settings
5. Click **Apply**

---

## Step 2: Configure WCCP on Router (Cisco Example)

```
ip wccp 80
ip wccp 443
!
interface GigabitEthernet0/1
  ip wccp 80 redirect in
  ip wccp 443 redirect in
```

On the WSG, configure the WCCP router address to match your gateway.

---

## Step 3: Configure Policy-Based Routing (Alternative)

For non-WCCP environments:

```
ip access-list extended WEB_TRAFFIC
  permit tcp any any eq 80
  permit tcp any any eq 443
!
route-map REDIRECT_TO_WSG permit 10
  match ip address WEB_TRAFFIC
  set ip next-hop [WSG IP]
!
interface GigabitEthernet0/0
  ip policy route-map REDIRECT_TO_WSG
```

---

## Step 4: SSL Inspection

For HTTPS transparent interception:

1. Enable SSL inspection in **Policy > SSL Inspection**
2. The WSG terminates the SSL connection and re-encrypts towards the destination
3. Clients must have the WSG CA certificate installed to avoid certificate errors
4. Sites in the SSL bypass list are passed through without inspection

---

## Step 5: Configure Bypass List

Some sites (banking, healthcare, VPN portals) should bypass SSL inspection:

1. Navigate to **Policy > SSL Bypass List**
2. Add domains/IPs that should not be inspected
3. Common bypass entries: `*.banking.com`, `*.gov.hk`, VPN services

---

## Verification

1. From a client, browse to `http://example.com`
2. Check **Logs > Web Logs** in UCSS for the request
3. Verify client IP is logged correctly
4. Test SSL inspection with `https://example.com`

---

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Traffic not redirected | Verify WCCP/PBR config on router |
| Routing loop | Check WSG default gateway doesn't redirect back |
| HTTPS certificate errors | Install WSG CA cert on clients |
| Bypass not working | Check bypass list entries and SSL policy order |
