## Overview

This page covers common issues encountered with the Nextguard Web Security Gateway and their solutions.

---

## Network Connectivity Issues

### Issue: No Internet Access After Inline Deployment
**Symptoms:** Clients cannot access the internet after installing WSG inline.

**Causes & Solutions:**
1. **Bridge interfaces not configured** → Verify eth0/eth1 are set as bridge interfaces in Network > Deployment Mode
2. **Incorrect cable connections** → Ensure LAN side connects to switch, WAN side connects to firewall
3. **Physical layer issue** → Check interface link status LEDs
4. **Fail-safe bypass engaged** → Check if bypass mode is enabled (traffic passes without inspection)

### Issue: Slow Web Browsing
**Symptoms:** Web pages load slowly after WSG deployment.

**Solutions:**
1. Disable SSL inspection temporarily to test if SSL processing is the bottleneck
2. Check CPU and memory usage in Dashboard > System Health
3. Enable hardware acceleration if available
4. Reduce the number of AV scan engines
5. Check upstream bandwidth — WSG is not the bottleneck

## SSL / HTTPS Issues

### Issue: HTTPS Sites Show Certificate Error
**Symptoms:** Users see "Certificate Not Trusted" errors on HTTPS sites.

**Solution:** Deploy the WSG CA certificate to client machines:
1. Download CA cert from **System > SSL Inspection > Download CA Cert**
2. Deploy via GPO: Computer Configuration > Windows Settings > Security Settings > Public Key Policies > Trusted Root Certification Authorities
3. Force GPO update: `gpupdate /force`
4. Restart browser

### Issue: Specific Sites Break After SSL Inspection
**Symptoms:** Certain websites (banking, healthcare) fail with SSL errors.

**Solution:** Add site to SSL bypass list:
1. Navigate to **Policy > SSL Bypass List**
2. Add the domain (e.g., `*.bankofamerica.com`)
3. Save and apply

### Issue: Certificate Pinning Errors
**Symptoms:** Mobile apps or desktop clients fail with certificate errors.

**Solution:** Add the app's domains to the SSL bypass list, or use IP-based bypass for known application IPs.

## Authentication Issues

### Issue: Users Repeatedly Prompted for Password
**Symptoms:** Every new browser window shows authentication prompt.

**Solutions:**
1. Verify NTLM/Kerberos is configured in **Authentication > Settings**
2. Ensure users are on a domain-joined machine
3. Check that the WSG hostname is in the **Trusted Sites** / **Intranet Zone** in IE/Edge settings
4. Verify SPN is registered correctly if using Kerberos

### Issue: AD Groups Not Appearing in Policy
**Solutions:**
1. Run manual sync: **Authentication > Group Sync > Sync Now**
2. Verify LDAP bind credentials have permission to read group membership
3. Check Base DN is correct and includes the OU containing the groups

## Policy Issues

### Issue: Policy Is Not Being Applied
**Symptoms:** Users can access blocked sites, or blocking happens when it shouldn't.

**Solutions:**
1. Check policy priority — lower number = higher priority
2. Verify user authentication is working (check logs for username)
3. Use **Policy Simulator** (Policy > Test Policy) to test a specific URL and user
4. Ensure policy is enabled and saved

### Issue: Block Page Not Appearing
**Symptoms:** Browser shows "connection refused" instead of block page.

**Solutions:**
1. Verify the block page server is running: **System > Block Page > Test**
2. Ensure clients can reach the WSG on port 8080 (or the configured redirect port)
3. Check if HTTPS interception is enabled for block page injection

## Performance Issues

### Issue: High CPU Usage
**Diagnostic steps:**
1. Navigate to **Dashboard > System Health**
2. Identify which process is consuming CPU (web engine, AV engine, DLP engine)
3. Temporarily disable DLP or AV to isolate the cause
4. Check for unusually high traffic volume from specific users

### Issue: Memory Exhaustion
**Solutions:**
1. Reduce log retention period
2. Clear old log files from **System > Storage Management**
3. Add RAM if hardware upgrade is possible
4. Consider load balancing with additional WSG appliances

## Diagnostic Tools

### Built-in Diagnostics
- **Packet Capture:** System > Diagnostics > Packet Capture
- **DNS Lookup:** System > Diagnostics > DNS Lookup
- **Ping / Traceroute:** System > Diagnostics > Network Test
- **Log Download:** System > Diagnostics > Download Logs

### Log Files (via CLI)
```bash
# Web traffic log
cat /var/log/trafficserver/traffic.out | grep ERROR

# Authentication log
tail -f /var/log/auth.log

# System log
journalctl -xe
```
