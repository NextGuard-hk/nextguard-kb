## What is the Web Security Gateway?

The Nextguard **Web Security Gateway (WSG)**, also known as **ASWG (Advanced Secure Web Gateway)**, is an enterprise-grade network security appliance that controls, monitors, and secures all web traffic between your users and the internet.

It serves as the central enforcement point for:
- Web access control and URL filtering
- SSL/TLS traffic inspection
- Malware and threat prevention
- Data Loss Prevention (Network DLP)
- Cloud application governance

---

## System Architecture

```
[Users / Endpoints]
       |
       v
[Nextguard WSG / ASWG]
   - SSL Inspection
   - URL Filtering
   - App Control
   - DLP Engine
   - AV Scanning
       |
       v
  [Internet / Cloud]
```

The WSG sits inline between your users and the internet, inspecting all traffic in real time.

---

## Core Components

### 1. Traffic Interception Engine
Handles network traffic capture and forwarding across all deployment modes (inline, proxy, transparent).

### 2. SSL/TLS Inspection Module
- Performs man-in-the-middle (MITM) decryption of HTTPS traffic
- Uses the WSG's CA certificate, deployed to endpoints via GPO or MDM
- Supports TLS 1.0 / 1.1 / 1.2 / 1.3
- SSL bypass list for banking and sensitive sites

### 3. URL Categorization Engine
- Database of 100+ URL categories
- Real-time cloud lookup for uncategorized URLs
- Custom allow/block lists
- Time-based access policies

### 4. Cloud App Control
- Identifies 5,000+ cloud applications
- Granular control (allow upload, block download, read-only)
- Shadow IT discovery

### 5. Network DLP Engine
- Integrated with Nextguard MDLP
- Inspects HTTP/HTTPS/FTP data for sensitive content
- Keyword, fingerprint, NLP, OCR detection methods

### 6. Antivirus & Malware Engine
- Dual-engine antivirus scanning (inline)
- Sandbox integration for unknown files
- Blocks malicious file downloads

## Deployment Modes

| Mode | Method | Client Config Required |
|------|--------|------------------------|
| **Inline** | Physical/logical inline between switch and firewall | None |
| **Explicit Proxy** | HTTP/HTTPS proxy address configured on clients | Yes (proxy settings) |
| **Transparent Proxy** | L4 redirect via firewall/switch policy | None |
| **Reverse Proxy** | Protects published internal web applications | None (server-side) |

> Inline mode is recommended for new deployments as it requires no client configuration.

---

## User Authentication

WSG supports multiple authentication methods:

| Method | Description |
|--------|-------------|
| **AD/LDAP** | Integrates with Active Directory via LDAP/LDAPS |
| **IP-based** | Assigns policies based on source IP address |
| **Kerberos SSO** | Transparent authentication for domain-joined clients |
| **Local Users** | Built-in user database for small deployments |
| **Captive Portal** | Browser-based authentication for guest/BYOD |

---

## Network Requirements

| Item | Requirement |
|------|-------------|
| **Management Interface** | Dedicated NIC for UCSS management |
| **Traffic Interfaces** | Minimum 2 NICs for inline mode |
| **DNS** | Access to internal and external DNS |
| **NTP** | Time sync required for SSL certificate validity |
| **UCSS** | Outbound HTTPS to UCSS management platform |

## Performance Specifications

| Model | Max Throughput | Concurrent Sessions | Users |
|-------|---------------|---------------------|-------|
| WSG-100 | 500 Mbps | 50,000 | Up to 500 |
| WSG-500 | 2 Gbps | 200,000 | Up to 2,000 |
