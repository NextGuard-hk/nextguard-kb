## Overview

In Reverse Proxy mode, the WSG sits in front of your internal web servers, intercepting inbound requests from external users. This protects internal applications from web-based attacks and provides centralized access control.

**Use cases:**
- Protecting web-based internal applications (OA, ERP, intranet portals)
- Enforcing authentication for published applications
- DLP scanning of inbound/outbound web application traffic

---

## Network Topology

```
[External Users]
       |
   [Internet]
       |
  [WSG - Reverse Proxy]    <-- Public IP / DNS points here
       |
  [Internal Web Server]    <-- Private IP
```

---

## Prerequisites

- [ ] WSG has a routable interface accessible from the internet (or DMZ)
- [ ] Internal web server IP and port known
- [ ] SSL certificate for the public domain (optional but recommended)
- [ ] DNS record points to WSG public IP

---

## Step 1: Enable Reverse Proxy Mode

1. Navigate to **Network > Deployment Mode**
2. Select **Reverse Proxy**
3. Configure the listening address and port (e.g., `0.0.0.0:443`)
4. Click **Apply**

---

## Step 2: Create a Virtual Host

1. Navigate to **Reverse Proxy > Virtual Hosts**
2. Click **Add Virtual Host**
3. Configure:
   - **Public Domain:** `app.yourcompany.com`
   - **Listening Port:** `443` (HTTPS)
   - **Backend Server:** `192.168.1.100:80` (internal server)
   - **SSL Certificate:** Upload or generate a certificate for the public domain
4. Enable **SSL Termination** if using HTTPS
5. Click **Save**

---

## Step 3: Configure Access Policy

1. Navigate to **Reverse Proxy > Access Policy**
2. Attach a policy to the virtual host:
   - **Authentication:** Require AD/LDAP login before accessing the app
   - **IP Restriction:** Limit access to specific source IPs
   - **DLP Scanning:** Enable content inspection on uploads/downloads
3. Save the policy

---

## Step 4: SSL Certificate Configuration

**Option A: Upload Existing Certificate**
1. Navigate to **Certificates > Import**
2. Upload the PEM/PFX certificate and private key
3. Assign to the virtual host

**Option B: Self-Signed (Testing Only)**
1. Navigate to **Certificates > Generate**
2. Generate a self-signed certificate for the domain

---

## Step 5: Test Access

1. From an external browser, navigate to `https://app.yourcompany.com`
2. Verify the page loads correctly from the internal server
3. Check **Logs > Access Logs** for the request
4. Verify SSL certificate is valid in the browser

---

## Troubleshooting

| Issue | Solution |
|-------|----------|
| 502 Bad Gateway | Backend server unreachable; check firewall and IP |
| SSL certificate error | Check certificate validity and domain match |
| Authentication not working | Verify LDAP/AD connection settings |
| Slow response | Check backend server performance; adjust timeout |
