# KB-终端替换证书方法

## 替换证书命令示例

使用 `createsetup.sh` 脚本生成安装包时，通过 `-e` 参数指定自定义证书：

```bash
bin/bash ./createsetup.sh -i <IP地址> -g /var/skyguard/... -e /var/skyguard/sps/program-data/setup/<证书路径>/cert.p12:<密码>
```

### 参数说明

- `-i`：如果客户使用域名方式生成终端安装包，此处应填写域名
- `-c 7`：禁用终端自我保护，没有该项时使用天空卫士默认设置
- `-e`：p12文件位置和密码，格式为 `p12文件路径:密码`

### p12文件说明

客户在自定义证书时，需要使用 2048 bit 的密钥长度。

## 证书替换流程

1. 先提申生成私钥 key
2. 找到客户的证书
3. 把签发的证书转换成 p12 格式
4. 终端生成安装包时使用转换好的证书

## 注意事项

- 替换证书时，最好直接让客户提供原来好的根证书
- 如果新生成证书，可能还是会偶发出现证书不受信任的问题
- 确保证书格式和密钥长度符合要求
