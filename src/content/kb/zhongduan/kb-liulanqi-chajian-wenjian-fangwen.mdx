# 浏览器插件与应用程序文件访问关系

## 功能对应关系表

| 应用程序文件访问 | 浏览器插件 | 效果说明 |
|---|---|---|
| 启用 | 禁用 | 跟普通进程文件访问一个效果 |
| 禁用 | 启用 | 有配置下载应用且勾选文件上传，如果浏览器访问的当前页面（Tab）匹配到下载应用的URL，则会对浏览器进程的文件操作进行策略匹配 |
| 启用 | 启用 | 应用程序文件操作都会进行策略匹配 |

## 下载应用终端实现逻辑

1. 浏览器插件向 EndpointClientAgent 上报当前浏览器正在访问的 URL
2. EndpointClientAgent 通过浏览器访问 URL 与下载应用列表进行匹配
3. EndpointClientAgent 接收到浏览器进程（pid）应用程序通道文件访问请求后：
   - 判断浏览器当前 URL 没有匹配上下载应用，且浏览器进程没有启用程序文件访问，则不做策略匹配
   - 判断浏览器当前 URL 没有匹配上下载应用，但浏览器进程启用了文件访问，则进行策略匹配
   - 判断浏览器当前 URL 匹配上了下载应用，如果检测目标对象中没有设置相关配置，则按默认策略处理
