# KB-事件正常，证据不存在

## 案例1：事件正常，证据不存在

### 排查思路

**第一步：**
先确定UCSS平台无法查询证据事件的事件、流量UUID

**第二步：**
进入证据目录查询对应的证据文件夹中是否有该证据文件。查看manager.log的报错信息，可能和Appliance相关。本文重点讨论证据文件确实不存在的情况。

### 正常证据文件入库查询方法

根据网络事件UUID进行搜索：
```bash
ls /var/skyguard/sps/forensics/incident/
# 例如: 723c8aaf4e8f.eml
# d84b2236-bf7c-4570-ae65-723c8aaf4e8f.eml
```

同时可以查看该时段incident.log的证据文件入库状态：
```bash
/opt/skyguard/sps/logs/sps-incident/incident.log
```

> **注意**：incident.log日志中如果grep UUID显示 `forensics file`，且证据文件后缀是eml，如果未grep到则证据文件不存在。

### 证据文件被删除的原因

证据文件不存在的车祸现场入库异常，原因是监控磁盘空间的monitor巡检周期为1个小时一次。

> **特别提示**：证据空间回滚的最小颗粒度是天为单位，也就是空间满时客户端只能显示当天的证据文件。原因是证据空间满时会按天回滚数据。

### 解密失败

如果出现 `javax.crypto` 相关报错，说明解密失败。
