# KB-证据空间遍历周期

## 定时删除证据文件的机制

关于各存储方式自动删除证据文件的机制：

### SPS服务

| 事件类型 | 遍历周期 | 备注 |
|---------|---------|------|
| 网络事件证据 | 每小时0分0秒执行 | 存储证据类型证据 |
| 终端事件证据 | 每小时10分0秒执行 | 检测并删除过期证据 |
| 移动事件证据 | 每小时20分0秒执行一次 | 最大空间复用的是网络证据空间大小 |
| 邮件事件证据 | 每小时40分0秒执行一次 | 3.11移除UCSS，目前未生效 |
| 发现事件证据 | 每小时50分0秒执行一次 | - |

## 为什么切换存储方式

以雅迪集团为例，17号从集中式切换到分布式：

- 网络证据大小：500GB
- 终端证据大小：1400GB

切换前，假设磁盘空间2TB，分布式存储。
切换后，Appliance服务接管存储：

1. Appliance服务采用网络证据空间使用大小
2. Appliance计算空间使用大小
3. 以redis的先进先出原则，按顺序删除
4. 从17号切换后开始
5. 因为在集中式下存储的旧数据已经迁移
