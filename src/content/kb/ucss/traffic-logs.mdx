# UCSS流量日志

## 概述

UCSS流量日志页面显示的日志数量始终保持 `200*N` 条（N=设备数）。流量日志主要用于故障排查，不需要留存大量的历史记录，但是需要很强的实时性。

当用户在界面点击“流量日志”时，将触发 `sps` 主动拉起 `spe` 获取流量日志。

## API流量日志堆积

堆积路径：

```
/var/skyguard/sps/traffic/temp/data/
```

如果ucwi上送的api流量日志json文件中queryID字段超长或者为空，会导致流量日志无法显示，或者sps-incident服务异常。

## 产生过程

1. 设备管理中所有网关设备，向它们的SPE模块发送API请求：

```bash
curl -X get -d '{"count":200}' https://<ip>:9443/spe/v1/getTrafficLog
```

每一台设备查询最新的200条流量日志。

2. 获取到的日志内容由 `sps-webmanager` 送入ES数据库中。

3. 测试spe是否正常产生流量日志：

```bash
cd /opt/skyguard/ucs/bin/
source ./env.sh
./spe_internal_util engine getTrafficLog
```

或者执行：

```bash
curl -X POST -d '{"count":200}' http://127.0.0.1:8888/spe/v1/getTrafficLog
```

## UCSS测定位方法

查看 manager.log 日志：

```bash
manager.log |grep getTrafficLog
```

## ES数据库中查看流量日志

查询ES中的流量日志数据：

```bash
curl -XGET http://127.0.0.1:9200/traffic* -H 'Content-Type:application/json' -d '{"aggregations":{"field":"detectionEngine","size":100}}'
```

## 分析状态字段

页面的分析状态对应字段：`analyze_status`

这个字段是页面实时抽取的，查看 `spe_access.log` 是看不出来的。

使用以下命令获取：

```bash
curl -X POST -d '{"count":200}' http://127.0.0.1:8888/spe/v1/getTrafficLog
```
