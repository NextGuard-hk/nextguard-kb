# 页面收集日志失败问题汇总

## 1、ucss管理日志收集超时

原因：定位时由于sps_client.py脚本中请求超时时间过短，导致日志收集超时失败。

位置：`/usr/lib/python2.7/dist-packages/sgcommon/`

### 解决方法

修改`set_timeout`函数中的timeout值，将默认值调大：

```python
def set_timeout(self, timeout=12):
    socket.setdefaulttimeout(timeout)
```

## 2、系统运行日志收集失败

发现客户环境中无法收集UCSS日志的根本是由于后台日志文件不存在。

### 解决方法

手工创建对应的文件后，系统日志收集恢复正常：

```bash
cd /var/log/skyguard/app
touch lcserver.log.202303020000
```
