# UCSS审批通知

## 重点

网络事件详情里面显示邮件通知发送成功，但是后台实际可能未发送。原因：发邮件是异步的，将邮件提交到发送队列就会显示成功。

## 邮件审批通知日志

日志路径：

```
/opt/skyguard/sps/logs/sps-incident/incident.log
```

### 常用日志检查命令

- `grep 事件uuid` - 查找证据文件入库状态、邮件审批状态
- `grep 1--事件id` - 查看是否有邮件审批通知发出
- `grep Notification@company-release.com` - 查找通知邮件
- `grep consuming` - 查询邮件通知耗时
- `grep deleted` - 查看是否有证据文件因为磁盘问题被删除
- `grep record` - 查看网络事件是否已入库

### 异常检查

- `grep 'email send failed'` - 检查邮件发送失败
- `grep 'network notification dispatcher'` - 检查网络通知分发器
- `grep 'restart next time'` - 表示未执行审批
- `grep '=== processing main retry ids:'` - 检查重试处理
- `grep 'notify retry send'` - 表示邮件服务重试发送
- `grep 'notification is null'` - 说明没有配置通知
