# SNMP无法监控到SMB挂载目录的空间

## 需求

客户想通过SNMP查询远程证据储存空间大小。

## 问题

执行 `snmpwalk -v2c -c skyguard 192.168.0.10 .1.3.6.1.4.1.2021.9.1.2` 发现无法到达这个分区。

## 解决办法

尝试先执行重启SNMP服务，这样应该会重新加载磁盘信息。

```bash
/etc/init.d/snmpd restart
```

重启完毕后，看看是否达到预期。如果依旧是不显示，请执行：

```bash
vi /etc/snmp/snmpd.conf
```

参考下图，新增：
```
disk /var/skyguard/sps/forensics 10%
```

再次查询，就有了。
