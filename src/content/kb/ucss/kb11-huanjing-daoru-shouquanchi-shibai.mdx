# KB11-环境导入授权池文件失败解决方法_V3.7.0

## 背景

设备池模式（wildcards）相比于传统的池模式，不需要绑定具体的device-id。

## 问题说明

TS Lab环境搭建V3.7.0 UCSS时，采用设备池授权方式，导入授权文件时提示“连接License服务器失败”。

公司现有2套licenseserver（内网、华为云），导入授权文件时会向云端授权服务器做校验，而当前licences服务器中查找不到该记录。

## 解决方案

因为V3.7.0版本有做Docker化处理，在修改UCSS配置时需要进入容器操作。

### 步骤1：SSH进入UCSS后台

切换为root账号，进入容器：
```bash
dcin sps
```

备选进入容器命令：`docker exec -it`
退出容器输入：`exit`

### 步骤2：修改容器中UCSS配置

将licenses文件验证授权服务器地址修改为正确的地址。

```bash
vim /opt/skyguard/tomcat/conf/c...
```

重启Tomcat：
```bash
service tomcat restart
```

### 备注

设备池模式授权时，必须连接SkyGuard授权服务器：
- `license.server.host=license.skyguard.com.cn`
- `license.server.host=license.skyguardmis.com`

修改配置后，授权池模式的文件可正常导入。
