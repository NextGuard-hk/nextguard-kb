## 审核与审批的区别

### 邮件审核

邮件审核是通用做法，主要有两种释放方式：

1. **直接回复邮件释放：** 通过mailto自动创建邮件包含释放码，调用邮件客户端直接发出
2. **点击同意释放：** 通过弹窗确认释放

### 审批流程（审批平台）

审批流程通过审批平台进行，支持多级审批：

- 分为 **一级审批** 和 **二级审批**
- 通道动作中的邮件隔离必须选择一级或二级审批
- 审批架里面需要给用户设置指定的审批员
- 需要在 **系统 > 邮件审批 > 邮件审批平台** 中启用

### 如何判断走审核还是审批？

> **重要：** 是否执行邮件审批，取决于策略通知里面是否勾选了审批员。如果没有勾选，则走审核流程。

## 通知模板配置

### 邮件主题模板

```
[UCSS事件： %事件ID%]-邮件待审核
```

### 邮件正文模板

模板支持以下变量：

| 变量 | 说明 |
|---|---|
| `%策略名称%` | 命中的策略名称 |
| `%事件时间%` | 事件发生时间 |
| `%通道%` | 触发通道 |
| `%文件名称%` | 相关文件名 |
| `%动作%` | 执行的动作 |
| `%来源%` | 来源信息 |
| `%登录名%` | 用户登录名 |

### 显示选项

- 显示LOGO
- 显示公司名称
- 显示动作
- 显示事件详情

## 审批平台登录

登录审批平台需要输入管理员邮箱地址，密码为AD用户密码。

## 多级审批流程

1. **无二级审批：** 点击同意/拒绝后，会弹窗直接释放
2. **有二级审批：** 一级审批后，会继续发邮件给二级审批员
3. **策略动作与审批：** 如果策略动作勾选了一级审批，则支持在审批页面进行审批
