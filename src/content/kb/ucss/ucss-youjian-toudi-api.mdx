# SEG邮件投递API

## 概述

UCSS提供API接口，允许第三方对指定的邮件进行重新投递。

## 用户场景

客户希望通过自己的其它平台（比如OA系统）对指定的邮件（比如隔离的邮件）进行重新投递。

## 接口说明

### 1. Check-in

进行check-in操作，获取access token。

**Request**

```
POST https://UCSS_URL/qkact/v0/checkin HTTP/1.1
Content-Type: application/json
Authorization: Basic base64("admin:password")
User-Agent: QKAct-External-Client

{
  "client-id": "5817AFB7-A263-43CA-..."
}
```

**Response**

```
200 OK HTTP/1.1
Content-Type: application/json

{
  "status": 0,
  "client-ip": "192.168.1.11",
  "access-token": "bcde6c8-73ec-11df-a...",
  "expiration": "Tue Oct 23 10:22:09 UTC 2023",
  "client-id": "5817AFB7-A263-43CA-..."
}
```

### 2. Check-out

进行check-out操作。

**Request**

```
DELETE https://UCSS_URL/qkact/v0/checkin/bcde6c8... HTTP/1.1
Content-Type: application/json
Authorization: Basic base64("client-id:access-token")
User-Agent: QKAct-External-Client
```

**Response**

```
200 OK HTTP/1.1
Content-Type: application/json

{
  "status": 0,
  "reason": "success",
  "access-token": "bcde6c8-73ec-11df-a...",
  "client-id": "5817AFB7-A263-43CA-..."
}
```

### 3. 邮件投递

进行邮件投递（加密投递仅后的版本支持）。

**Request**

```
POST /qkact/v1/seg/email/delivery HTTP/1.1
Content-Type: application/json
Authorization: Basic base64("client-id:access-token")
User-Agent: QKAct-External-Client

{
  "smtpId": {
    "value": ""
  },
  "encrypt": true  // true表示加密投递
}
```

**Response**

```
200 OK HTTP/1.1
Content-Type: application/json

{
  "status": 0,
  "reason": "成功"
}
```
