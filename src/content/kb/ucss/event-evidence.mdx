# UCSS事件证据管理

## 概述

- `incident` 服务内部配置了30个线程池，最多支持30个线程同时处理事件。
- 证据文件加密方式为64位对称加密算法，每个证据都是有唯一密钥的。
- 系统默认禁止root与远程访问证据服务。
- sps把reids临时数据入库到ES，每次处理约10条信息（json），3.3.0-HF103版本后优化为每次处理20个json。

## 证据存储路径

### UCSS证据路径

```
/var/skyguard/sps/forensics/incident  #证据文件存储
/var/skyguard/forensics/encrypted     #分布式存储加密证据
```

### 网关证据/事件文件待上传路径

```
/opt/skyguard/ucs/var/spe/incident_log/forensics
/opt/skyguard/ucs/var/spe/incident_log/incident
/opt/skyguard/ucs/var/spe/incident_log.error/
```

### 终端事件/证据待上传路径

```
/opt/skyguard/ucs/var/dm/incident
/opt/skyguard/ucs/var/dm/forensics/
```

### ITM日志堆积

```
/opt/skyguard/ucs/var/dm/itm_log
```

## 日志查看与排查

主要日志文件：

```bash
/opt/skyguard/sps/logs/sps-incident/incident.log
```

常用grep命令：

```bash
# 查看事件uuid、证据文件入库状态、邮件审批通知
grep 事件uuid incident.log

# 查看是否有邮件审批通知发出
grep 1--事件id incident.log

# 查看证据文件是否被删除
grep "has been deleted" incident.log

# 查看网络事件是否入库
grep record incident.log

# 查看邮件通知耗时
grep consuming incident.log

# 检查证据空间是否超出
grep overweight incident.log
```

表示收到证据：

```
Incident forensics file received from dlp server
```

表示收到事件：

```
Incident data file received from dlp server
```

## 开启调试日志

```bash
/var/skyguard/sps/program-data/advanced/log.cfg
# 修改日志级别为DEBUG
```

## 增加内存配置

后台增加sps-incident内存，减轻redis堆积，提高ES入库效率：

```bash
/etc/init.d/sps-incident  # 增加 memory_config=4096
```

## 事件处理流程

SPE把网络事件和证据文件通过appliance建立的通道发送到UCSS平台进行处理和存储。

## 网络事件搜索限制

终端或者网络事件中对源或目标进行搜索时，筛选用户数量最大支持1000，大于1000后的不会显示。ES最大查询上限是1024，PM预设最大不能超过1000。
