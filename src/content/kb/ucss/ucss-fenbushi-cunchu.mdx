# 分布式存储

## 分部署存储流程

1. SPE的证据是送给APP直接保存在本地，并且证据落地后会更新事件中的证据路径
2. UCSS只负责入库事件，事件中会更新事件对应分布式存储节点的证据路径

## 案例一：分布式存储终端事件证据不存在

查看日志：
```
/opt/skyguard/sps/logs/sps-incident/incident.log
```

如果出现 `Duplicate key` 错误，表示终端事件UUID重复（终端事件lite分类）。

### 解决办法

事件太多，不太好找是哪些事件UUID重复，但是重复的事件会导致后续事件入库失败。

排查步骤：
1. 查incident.log日志，如果还有Duplicate key字段报错，执行以下命令
2. 连接Redis：
```bash
redis-cli -n 1
```
3. 清理Redis堆积：
```bash
ltrim incident_distribute 10 -1  # 删除redis堆积的前10条数据
```

> **备注**：redis执行是删除操作后，事件不会丢，但是对应的证据可能需要重新上传。
