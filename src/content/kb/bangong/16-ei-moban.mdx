# 开EI模板

## 标题格式

样例：【海尔集团 UCSS-SPS-3.10】主UCSS和备UCSS过滤规则不同步

## EI必填字段

### 【环境】
- 终端版本信息
- UCSS版本信息
- 补丁情况

### 【紧急度和背景】
注：如果是紧急的注明为什么紧急的背景

### 【受影响范围】

### 【故障现象描述】

### 【TS故障排查步骤】内部能否复现

### 【期望的结果和现实的结果】

### 【需要开发帮助的内容】

### 【日志情况】
1. Debug Level
2. 现象发生的时间
3. 相关的文件名和事件截图
4. 下载地址

## EI模板字段说明

| 字段 | 说明 |
|------|------|
| 用户信息 | 项目名称，如：平安集团 |
| 用户数量 | 如：3000用户 |
| 当前版本 | 如：1.3.5.2，包括打过的Hotfix |
| 优先级 | 需要TS自己来填，如果在POC一律都是最高优先级 |
| 网络流量 | 如：500Mbps |
| 网络环境 | 可选项，如果有更好，如果是部署的问题则必填 |
| 部署模式 | 旁路/串行/代理/终端/UCWI |
| 所属模块 | 该问题出现的设备类型，如：GatorCloud/DSG/Hybrid SWG/UCSS Lite/Hybrid UCWI |
| 设备型号 | 如：51000/11000/5100/1100 |
| 设备序列号 | 序列号未来要做为让TS支持的必选项 |
| 过期时间 | 过期时间未来要做为让TS支持的必选项 |
| 是否通过Hotfix解决 | 默认是放在下一版本修复，如需Workaround |

## 问题描述要求

1. 详细描述问题现象说明、复现步骤、截图、日志
2. 实际结果，期望结果，出现频率，期望解决在哪个版本
3. 如果是随机问题则需要标明问题出现的时间点，便于查日志
4. 目前已经做过了哪些尝试性操作，如：更改后台配置

## 提BUG与提功能区分

- **提BUG**: 红+绿
- **提功能**: 红+蓝
