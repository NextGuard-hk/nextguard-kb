# Keepalived无法启动问题定位

## 日志路径

```bash
/var/log/syslog |grep VRRP |grep Keepalived
```

## 3.15/3.16版本路径

```bash
/var/log/skyguard/app/app.log
```

如果没有app.log，应该是早期版本的权限问题，解决办法：

```bash
chown root:syslog /var/log/skyguard/app/
chmod 775 /var/log/skyguard/app
```

## keepalived.log

如果log里不输出内容，可能是rsyslog配置问题：

```bash
vi /etc/rsyslog.d/keepalived.conf
```

确认配置内容为：

```
local0.* /var/log/keepalived.log
```

然后重启rsyslog：

```bash
service rsyslog restart
```
