# ITM常用故障处理

## 判断ITM是否在正常接受数据

### 抓包检查
在ITM上抓514端口的包，确定NCP或者UCSS等设备有正常发送数据。

### 查看日志
```bash
cd /var/log/itp/itpserver/
tail -f itm_debug.log
```
检查UCSS是否正常把终端日志发送到ITM。

## 判断ES数据库连接是否正常

```bash
netstat -anpt |grep 9200
curl http://localhost:9200/_cat/indices
```
检查ES索引是否正常创建和更新。

## 进入Kafka容器

```bash
docker exec -it b61d94e2fd17 /bin/bash
```
检查Kafka消息队列是否正常运行。

## ITM服务重启

```bash
systemctl restart itpserver
systemctl status itpserver
```

## 日志文件路径

| 日志类型 | 路径 |
|---------|------|
| ITM调试日志 | /var/log/itp/itpserver/itm_debug.log |
| ITM错误日志 | /var/log/itp/itpserver/itm_error.log |
| 初始化日志 | /var/log/itp/itpcli.log |
| UWSGI日志 | /opt/skyguard/itpserver/logs/uwsgi.log |

## 配置文件路径

| 配置文件 | 路径 |
|---------|------|
| ITM API配置 | /opt/skyguard/itpserver/itmapi_uwsgi.ini |
| Docker Compose | /opt/skyguard/itpcompose/docker-compose.yml |
| PID文件 | /var/run/itpserver.pid |
