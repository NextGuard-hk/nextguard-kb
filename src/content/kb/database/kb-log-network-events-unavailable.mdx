# KB-日志和网络事件无法查看，提示 None of the configured nodes are available

## 问题描述

CSS平台中无法查看邮件日志和DLP事件，提示“None of the configured nodes are available”。

## 排查步骤

### 1、检查ES内存是否过低

一般需要2-4G以上。

### 2、查看Elasticsearch日志

```bash
/var/log/elasticsearch/elasticsearch.log
```

日志中可能出现 `TransportSearch` 相关错误。

### 3、健康度检查

```bash
curl http://localhost:9200/_cat/health?v
```

发现状态是yellow，结合日志看，red状态时ES不可用，yellow时部分功能受限。

## 解决方案

### 1、删除ES死掉生成的dump文件

```bash
du -sh /var/lib/elasticsearch/*
```

删除 `java_pid*.hprof` 文件（每个约3.7G）：

```bash
rm -f /var/lib/elasticsearch/java_pid*.hprof
```

### 2、定位发生yellow的索引

```bash
curl http://localhost:9200/_cat/indices | grep yellow
curl http://localhost:9200/_cat/indices | grep red
```

查看待归档的文件大小是否为0，如connect-log类型的索引，如果当时本身没有数据，可以删除。

### 3、手动删除无数据索引

删除大小为0的connect-log索引后，再次检查健康度：

```bash
curl http://localhost:9200/_cat/health?v
```

确认状态变为green后，问题解决。

### 4、验证日志服务恢复

```bash
curl http://localhost:9200/_cat/health?v
curl http://localhost:9200/_cat/indices
```

确认ES状态为green，日志和网络事件可正常查看。
