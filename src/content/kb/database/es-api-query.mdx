# Elasticsearch 接口查询

## 说明

所有的查询语句都是Elasticsearch原生支持的语句，如果需要更多的查询，可以直接查询Elasticsearch的官方文档。

## 查询当前所有索引

```bash
curl http://localhost:9200/_cat/indices;
```

结果示例：第三列即为索引的名称。

## 如何确定索引名称

进入管理平台：**系统-归档** 页面，可以查看到当前包含了哪几类事件日志索引。

## 查看状态为活跃的分区名称

分区名称：Dlp-20210401-01（以实际的系统为准）

说明：当前索引中，凡是日期20210401-01结尾的都是当前活跃的索引，新的事件都会存在此索引中。

分区名称和后台es索引名称的对应关系如下：

- DLP事件：以dlp开头，日期结尾，中间加上事件类型
- 网络事件：dlp-network-20210401-01
- 终端事件：dlp-endpoint-20210401-01
- 发现事件：dlp-discovery-20210401-01

如果经过了数次归档（默认三个月），每种类型的索引会有多个，以不同的日期结尾。以终端事件的索引为例，可能会有：
- dlp-endpoint-20210401-01
- dlp-endpoint-20210101-01
- dlp-endpoint-20201001-01

## 根据条件查询终端事件

首先确定索引名称，例如查询终端事件，索引名称为：dlp-endpoint-20210401-01

查询此索引下的所有终端事件：

```bash
curl http://localhost:9200/dlp-endpoint-20210401-01/_search?pretty
```

支持模糊查询，例如可以查询所有终端索引下的事件：

```bash
curl http://localhost:9200/dlp-endpoint-*/_search?pretty;
```

## 根据高级筛选条件来查询

### (1) 根据检测时间精确查询

> 日期格式要严格按照此格式

```bash
curl http://localhost:9200/dlp-endpoint-*/_search?pretty -d '{"query":{"match":{"detectDateTime":"2021-05-12T02:50:28.993Z"}}}' -H "Content-Type:application/json"
```

### (2) 根据检测时间范围查询

```bash
curl http://localhost:9200/dlp-endpoint-*/_search?pretty -d '{"query":{"bool":{"must":[{"range":{"detectDateTime":{"from":"2021-05-09T16:00:00.000Z","to":"2021-05-12T15:59:59.999Z","include_lower":true,"include_upper":true,"boost":1.0}}}]}}}' -H "Content-Type:application/json"
```

### (3) 按照事件时间查询

```bash
curl http://localhost:9200/dlp-endpoint-*/_search?pretty -d '{"query":{"match":{"incidentDateTime":"2021-05-12T02:50:28.993Z"}}}' -H "Content-Type:application/json"
```

### (4) 按照事件时间范围查询

同(2)语句，查询语句中的 detectDateTime 替换为 incidentDateTime。

### (5) 按照主机名查询

同(1)查询语句中的 detectDateTime 替换为 hostname。

```bash
curl http://localhost:9200/dlp-endpoint-*/_search?pretty -d '{"query":{"match":{"hostname":"test"}}}' -H "Content-Type:application/json"
```

### (6) 按照安全级别查询

severityType 中必须是数字，代表的意思可以参考段落4中的字段说明。

```bash
curl http://169.254.253.5:9200/dlp-endpoint-*/_search?pretty -d '{"query":{"bool":{"must":[{"terms":{"severityType":[1,2,3,4],"boost":1.0}}]}}}' -H "Content-Type:application/json"
```

## 接口查询返回字段说明

以终端事件的结构为例说明如下：

| 字段 | 说明 |
|------|------|
| took | 查询耗时 |
| timed_out | 是否超时 |
| _index | 终端事件所属的索引名称 |
| _type | endpoint-incident，终端事件所属的索引类型 |
| _id | 终端事件所属的索引的uuid |
| serialId | 事件ID |
| transactionId | 流量UUID |
| trafficId | traffic ID，内部用，业务上不需要关心 |
| incidentType | 事件类型，3代表是终端事件 |
| detectDateTime | 检测时间：utc时间，需要+8小时才是页面上显示的中国时间 |
| detectTime | 检测时间的小时数：utc时间 |
| incidentDateTime | 事件时间：即此条记录进入es的时间 |
| incidentTime | 事件时间的小时数 |
| severityType | 安全级别：1-高 2-中 3-低 4-信息 |
| statusType | 事件状态：1-新 2-进行中 3-关闭 4-误报 5-提权 |
| actionType | 动作：2-阻止，1-允许 |
| policyAction | 策略动作 |
| agentAction | agent动作，内部用，不需要关心 |
| isIgnored | 忽略状态：false-未忽略 true-忽略 |
| breachContents | 命中内容：命中的策略中的规则名称，如关键字、正则等等 |
| maxMatches | 最大匹配：命中规则的最大匹配次数 |
| transactionSize | 流量大小：命中的此条流量的大小 |
| deployVersion | 部署版本，内部用 |
| detectAgentUuid | 检测引擎UUID |
| detectAgentType | 检测引擎类型：SWG lite 2:ICAP 3:DSA 4:MTA 5:Print 6:Endpoint 7:UCWI 8:PCAP 9:MOBILE 10:EMAIL BACKTRACKING |
| detectAgentHostname | 检测引擎的主机名 |
| detectAgentIp | 检测引擎的IP |
| detectAgentVersion | 检测引擎的版本 |
| analyzeAgentUuid | 分析引擎的UUID |
| analyzeAgentType | 分析引擎的类型：100-CAE |
| analyzeAgentHostname | 分析引擎所在的主机名 |
| hierarchy | 分级信息，默认为0，只有启用了管理平台的分级功能才有其他值 |
| isOrdinary | 是否是普通的事件，不开启itm，不需要关心 |
| totalMatches | 是否是普通的事件 |
| matchedPolicies | 匹配的策略数组 |
| isTrickle | 是否零星式策略，默认false |
| isVisible | 是否可见 |
| isItmPolicy | 是否是零星式策略，默认false |
| elementType | 匹配的规则中的元素类型：1:正则 2:字典 3:外部脚本 4:文件类型组 5:关键字 6:脚本 7:文件指纹 8:机器学习 9:终端位置 10:文件名称 11:附件数量 12:数据库指纹 13:文件大小 14:二进制 15:压缩文件深度 16:加密文件 17:格式不匹配文件 18:内置模板 20:数据分类 21:标签 |
