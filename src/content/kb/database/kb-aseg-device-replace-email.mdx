# KB-ASEG设备替换或者迁移后，无法获取SEG设备上的邮件原文

## 问题描述

UCSS上的邮件日志是根据ASEG的 `device_id` 查询的。设备替换或迁移后，device_id发生变化，导致无法获取旧设备上的邮件原文。

## 解决方法

通过修改ES中邮件日志的device-id字段，将旧设备ID替换为新设备ID。

### 1. 尝试只改一条邮件日志数据

三个变量：“旧deviceID” “新deviceID” “新deviceID值”

```bash
curl -XPOST 'http://169.254.253.5:9200/email-message-*/_update_by_query' -H 'Content-Type: application/json' -d '
{"script":{"source":"if(ctx._source.deviceId==\'旧deviceID\'){ctx._source.deviceId=\'新deviceID\'}"},"query":{"terms":{"deviceId":["旧设备UUID"]}},"boost":1.0},"track_total_hits":2147483647}'
```

### 2. 单条日志改完没问题后，批量修改邮件日志device-id

```bash
curl -XPOST 'http://169.254.253.5:9200/email-message-*/_update_by_query' -H 'Content-Type: application/json' -d '
{"script":{"source":"if(ctx._source.deviceId==\'旧设备UUID\'){ctx._source.deviceId=\'新设备UUID\'}"},"query":{"bool":{"must":[{"terms":{"deviceId":["旧设备UUID"]}},{"boost":1}]}},"track_total_hits":2147483647}'
```

> 注意：测试环境测试可以，但需注意在生产环境谨慎操作。
