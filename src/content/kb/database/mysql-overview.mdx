# 04.MYSQL

UCSS的所有版本中都包含mysql，终端标签相关数据存储在mysql中，其中容器化后，MySQL单独运行在容器中了

查看mysql版本：
```
mysql --version
```

日志路径：
```
/var/log/mysql/mysql.log
```

---------------------------------------------

mysql.log日志默认不开启，开放方法
```
vi /etc/mysql/my.cnf  #取消下述两行注释，重启mysql即可
#general_log_file= /var/log/mysql/mysql.log
```

3.3版本ucss中mysql主要时标签服务，mysql.error日志路径：
```
/opt/skyguard/ucs/var/lms/mysql/log
```

3.3版本mysql的监听端口是8859
```
netstat -napt |grep 8859
tcp        0      0 127.0.0.1:8859          0.0.0.0:*               LISTEN      52192/mysqld
```

## 3.16变成great了

3.3UCSS进入mysql方法：
```
/opt/skyguard/ucs/bin/mysql -h 127.0.0.1 -P 8859 -u skyguard -pskyguard123
```

对应配置文件：
```
/opt/skyguard/ucs/etc/lms_local.xml
          <db_user>skyguard</db_user>
          <db_password>skyguard123</db_password>
```

3.3 ASEG/DSG/ASEG mysql登录相关信息

未加固mysql密码前（ats-054版本前）
```
mysql -usegapi -psegapi #ASEG
mysql -ucert_server -pcert_server  #DSG/ASWG
```

加固mysql密码后（ats-054），seg/dsg/swg同一用一个账号密码
```
mysql -usecgator -pSECGATOR@SkyGuard405.3
```

容器化后的登录信息：
```
mysql -uroot -pMysql@SkyGuard253.3
```
