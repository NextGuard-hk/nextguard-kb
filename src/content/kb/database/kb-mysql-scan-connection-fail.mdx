## KB-mysql数据库扫描报，连接数据源失败

目前数据库扫描不支持MySQL8.0以上版本。

报错日志示例：

```
2022-03-23 10:32:41,645 [0x7f5f504f5700] <DSA 10.13.4.72, port: 3306 successfully.
2022-03-23 10:32:41,645 [0x7f5f504f5700] <DSA <10.13.4.72> successfully, ip is: <10.13.4.72>
2022-03-23 10:32:41,649 [0x7f5f504f5700] <DSA [unixODBC][MySQL][ODBC 5.3(w) Driver]Authentication plugin 'caching_sha2_password' cannot be loaded: /usr/local/mysql/lib/plugin/caching_sha2_password.so: cannot open shared object file: No such file or directory QODBC3: Unable to connect
```

### 原因

mysql8之后的版本，加密默认是 `caching_sha2_password`，系统镜像中没有安装加密的依赖，无法使用 `caching_sha2_password` 认证方式。

### 解决办法

创建一个新的用户，创建的时候使用 `mysql_native_password` 认证方式：

```sql
ALTER USER 'yourusername'@'localhost' IDENTIFIED WITH mysql_native_password BY 'yourpassword';
```
