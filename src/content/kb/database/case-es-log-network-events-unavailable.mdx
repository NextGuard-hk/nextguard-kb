# 15、日志和网络事件无法查看，提示 None of the configured nodes are available

## 问题描述

UCSS平台中无法查看邮件日志和DLP事件，但是过一段时间后可能会恢复，该现象反复出现。

## 原因分析

邮件日志和网络事件与ES数据库有关，查看ES日志发现报分片失败错误（TransportSearchAction）。

检查ES日志：

```bash
cat /var/log/elasticsearch/elasticsearch.log
```

健康度检查：

```bash
curl http://localhost:9200/_cat/health?v
```

发现状态为yellow，结合日志看，red状态时ES不可用，yellow状态勉强可用。

## 解决方案

### 1、删除ES死掉生成的dump文件

免得磁盘爆掉：

```bash
du -sh /var/lib/elasticsearch/*
```

删除大量的 `java_pid*.hprof` 文件，检查磁盘空间：

```bash
df -h
```

### 2、定位发生yellow的索引

```bash
curl http://localhost:9200/_cat/indices | grep yellow
curl http://localhost:9200/_cat/indices | grep red
```

查看connect-log类型的索引，如 `connect-log-20181109-01`、`connect-log-20190519-02`、`connect-log-20180810-01` 等，这几个文件大小为0，是待归档的文件。

应该是当时本身没有数据，可以删掉。推测是这几个空文件导致查询失败（正常的逻辑是归档成功就会删掉这个index）。

### 3、手动删除无数据索引后验证

手动删除后，检查健康度：

```bash
curl http://localhost:9200/_cat/health?v
```

确认状态变为green，问题解决。同时检查ES日志确认没有新的错误：

```bash
tail -f /var/log/elasticsearch/elasticsearch.log
```
