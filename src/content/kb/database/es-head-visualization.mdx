### 2. 安装es-head到浏览器扩展程序中

### 3. 在浏览器中开启es-head

但在连接测试时失败，没有获取到ES中的任何信息，同时，在本机上telnet ES服务器的地址+9200端口都是不通的！

**问题原因：**
在ES默认的配置文件中，从安全性考虑，是不允许从远程主机直接连接到ES的9200端口的，默认只能从安装了ES服务的本机进行连接。如果要通过远程直接连接9200端口，需要修改后台配置文件。

### 4. 在UCSS后台修改ES配置文件

```bash
# vi /etc/elasticsearch/elasticsearch.yml
```

修改如下两个地方：
1. 开启且修改： `network.host: 0.0.0.0` （也可以修改为远程主机具体的IP地址）
2. 开启： `http.port: 9200`

之后重启ES服务；

```bash
# /etc/init.d/elasticsearch restart
```

但发现ES服务重启失败，服务起不来了。

```
root@ucss:/etc/elasticsearch# /etc/init.d/elasticsearch status
[FAIL] elasticsearch is not running ... failed!
```

**故障原因：**
由于修改了默认配置，需要增加如下配置后，ES服务才能起来：
在最后增加如下配置： `bootstrap.system_call_filter: false`

之后重启ES服务；

```bash
# /etc/init.d/elasticsearch restart
root@ucss:/etc/elasticsearch# /etc/init.d/elasticsearch status
[ ok ] elasticsearch is running.
```
