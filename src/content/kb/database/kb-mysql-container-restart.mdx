## mysql容器不断重启/ats容器不断重启/推送license到后台失败

### 背景

客户反馈导入授权时，提示：推送license到后台失败。

### 排查过程

1. 发现ats容器 apache2无法启动
2. 发现ats容器每隔几分钟重启
3. 发现mysql容器一直在重启

### 初步结论

ats容器反复重启，导致apache2一直restarting中。看起来是因为mysql没有正常完成初始化。

查看事件日志及查看mysql日志：

```bash
vi /var/log/skyguard/mysql/mysql_error.log
```

如果mysql启动异常，还可以通过以下命令调试：

```bash
docker start -i appliance_mysql_1
```

### 解决办法

1. Mysql的数据文件在主机的 `/var/skyguard/mysql/` 目录下，需要清理
2. 还要删除掉主机中的 `/etc/ats_etc/seg/first_boot*` 文件（没有就不用管）
3. 重新初始化，客户环境初始化成功的文件只有两个
4. 重启 mysql 容器：

```bash
docker restart appliance_mysql_1
```

另外还配合seg策略同步失败的问题，重新初始化并注册到ucss即可。
