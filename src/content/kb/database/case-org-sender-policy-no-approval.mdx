# 案例6、组织机构发件人命中策略后，UCSS未给主管发送邮件审批

## 问题描述

组织机构用户里的某个用户发邮件命中策略后，邮件命中 DLP 测试关联发送主管邮箱正常，但实际未发送审批邮件。

## 原因分析

dsg-mta 匹配发件人的 uuid，并把这个 uuid 匹配成员的主管信息，导致无法成功给主管发送邮件。

目前组织机构中只有这一个人（或者少数）有次问题，以为是个例，后台没生效。

针对组织机构中手动添加的主管信息，会造成主表和关系表冗余。已在 3.3 上修复，改为统一读取 sps-059 补丁即可。

## 解决方案

手动修复办法，进入 pg 数据库：

### 1. 查找主管关联信息

```sql
select manager_id from system.udt_directory_customuser_manager
where uuid = '对应用户uuid';
```

### 2. 更新主管关联

```sql
update system.udt_directory_entries set manager = '主管uuid'
where uuid = '用户uuid';
```

### 复现方法

1. 查看 SQLite3 ORS.db 中的 Custom_user 表

```sql
select * from Custom_user where common_name = '用户名';
```

2. 进入 pg 数据库，删除该用户对应的 manager uuid，因为 sps-059 后不根据这个表关系选择主管

```sql
update system.udt_directory_entries set manager = '' where uuid = '用户uuid';
```

> 说明：已在 3.3 版本通过 sps-059 补丁修复，新的补丁统一从主表和关系表读取主管关联。
