# 数据库弱密码/UCSS平台修改PG数据库密码方法

## 修改pg数据库中sps用户的密码

```bash
# 修改sps用户密码
psql "host=pg所在机器的ip dbname=sps_db user=sps password=sps" \
  --command "alter user sps with password '新密码';"

# 示例：修改为Admin@1234
psql "host=172.26.33.37 dbname=sps_db user=sps password=sps" \
  --command "alter user sps with password 'Admin@1234';"

# 示例：修改为SkyGuard%123
psql "host=172.26.33.37 dbname=sps_db user=sps password=sps" \
  --command "alter user sps with password 'SkyGuard%123';"

# 修改postgres用户密码
psql "host=127.0.0.1 dbname=sps_db user=postgres password=postgres" \
  --command "alter user postgres with password 'Admin@1234';"

# 另一种方式
psql "host=127.0.0.1 dbname=sps_db user=sps password=Admin@1234" \
  --command "alter user sps with password 'Admin@2345';"
```

## 修改配置文件

修改以下配置文件中的 `postgre.jdbc.password=新密码`：

```
/opt/skyguard/tomcat/conf/config.properties
/opt/skyguard/sps/apps/sps-incident/config/incident.properties
/opt/skyguard/sps/apps/sps-webservice/config/webservice.properties
```

修改初始化脚本：

```
/opt/skyguard/sps/dataInstall/init-data.sh  # 第32行
passWord="新密码"
```

## 3.3高可用环境还需修改配置文件

```bash
vi /opt/skyguard/sps/monit/check_pem_status.sh
# 编辑11行，删除 "=sps" 后保存

vi /opt/skyguard/sps/monit/check_approval_status.sh
# 编辑14行，删除 "=sps" 后保存

# 然后重启monit
/etc/init.d/monit restart
```

## 重启服务生效

```bash
/etc/init.d/tomcat restart
/etc/init.d/sps-incident restart
/etc/init.d/sps-webservice restart
/etc/init.d/sps-pem restart
/etc/init.d/sps-mail-approval restart
```
