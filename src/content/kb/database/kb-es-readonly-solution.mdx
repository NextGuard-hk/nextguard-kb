# KB-ES只读解决办法

## ES只读

一旦磁盘data空间超过85%后，ES数据库会变成只读锁表，系统将无法写入新的事件数据。

> 备注：3.10的后续版本磁盘空间是否后，es会自动解锁

## 判断是否只读

```bash
curl -s http://127.0.0.1:9200/_settings?pretty
```

查看返回结果中：
- `max_thread_count` : "1"
- `read_only_allow_delete` : "true"
- `max_thread_count` : "1"

看到true表示只读。

## 老式判断方法

```bash
curl -H "Content-Type: application/json" http://127.0.0.1:9200/_all/_settings
```

在输入的结果中随便选择一个进行查询，因为要锁所有表都会被锁：

```bash
curl -H "Content-Type: application/json" http://127.0.0.1:9200/<index_name>/_settings
```

如果在查询的结果中有 `read_only_allow_delete` 字段，并且值为true：

```json
"block" : {
  "read_only_allow_delete" : "true"
}
```

结果为true，就是ES被锁了。

## 解锁方法

```bash
curl -XPUT -H "Content-Type: application/json" http://127.0.0.1:9200/_all/_settings -d '{"index.blocks.read_only_allow_delete":null}'
```

或者：

```bash
curl -XPUT -H "Content-Type: application/json" http://127.0.0.1:9200/<index_name>/_settings -d '{"index.blocks.read_only_allow_delete":null}'
```

## 解锁后异常处理

如果解锁后异常可参考：

1. `service sps-incident stop` — 没有新的事件产生时执行
2. ES异常时，删除es的pid位置，在 `/var/run/elasticsearch/` 目录下
