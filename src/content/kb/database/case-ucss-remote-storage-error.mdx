# 案例2、UCSS保存远程存储时页面报内部错误但后台已挂载成功

## 问题描述

用户挂载了一个 110T 的 NAS 做为远程证据文件空间，后台可以成功挂载配置，依然时本地存储，再次保存远程配置时，报“服务器内部错误”，提示证据文件空间不足。

存储设置路径：系统 > 基本设置 > 存储设置

- 证据文件/邮件存储方式：集中存储
- 存储位置：远程
- 共享类型：NFS (Network File System)

## 原因分析

UCSS进行证据挂载时，后台由 app 组件完成挂载动作。app 进行 NAS 挂载时，会遍历所有文件和文件夹进行证据文件扫描和赋权。

由于挂载赋权时间太久，导致 app 执行赋权太久，sps得不到返回结果，因此报错服务器内部错误。

统计文件遍历赋权耗时约 7 分钟。

根本原因：NAS 里文件太多，短时间内无法完成空间统计。

## 解决办法

针对大容量 NAS（如 110T），一旦挂载成功后，需要手动通过数据库更新配置：

### 更新 UCSS 页面数据库，将页面配置成远程 NAS

如果页面保存远程存储失败，但后台已挂载成功，可通过数据库直接修改：

```sql
-- 查看当前存储配置
sps_db=> select json_info from system.global_configuration where key='STORGE_SETTINGS';
```

将结果复制下来，把 `"locationType":0` 改为 `"locationType":1`，然后执行更新：

```sql
update system.global_configuration set json_info='...修改后的json...' where key='STORGE_SETTINGS';
```

> **说明：** `locationType` 值为 1 代表远程，0 代表本地。
