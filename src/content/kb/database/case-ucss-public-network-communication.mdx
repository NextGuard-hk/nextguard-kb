# UCSS如何和公网的UCSSlite或者其他网关通信

## 问题场景

UCSS在公司内网，UCSSlite/ASEG等网关设备在公网或DMZ等区域，导致UCSS无法直接和网关通信。客户调整网络做了NAT映射后，保证UCSS和UCSSlite的9443/8443/8447等端口均可以互通，且lite也成功注册到了UCSS上，但是设备列表界面，UCSSlite显示离线。

## 原因分析

造成设备监控页面显示设备离线的根本原因是：SPS发起设备状态请求或策略下发时，用的是PG数据库中 `system.device_info` 表中设备的真实注册IP。

## 解决办法

修改 `system.device_info` 中lite的真实IP为NAT后的IP地址，保证网络可达。

```sql
sps_db=> update system.device_info set device_ip='172.22.80.243'
where device_ip='192.168.0.193';
UPDATE 1
```

执行后，设备监控页面，lite应该可以正常上线。

如果依旧未上线，可以查看SPS的 `manager.log` 日志，可能会看到类似报错：

```
192.168.0.193:9443 [/192.168.0.193] failed: Connection timed out;
```

这是因为tomcat调用的是缓存中lite的IP地址，重启tomcat后，SPS再次请求lite时，就会按照PG数据库中指定的IP进行了。

## 修改后验证

1. 设备状态正常
2. 策略同步正常
3. 日志事件上传正常

## 注意事项

后续lite如果执行了注册操作，PG数据库中的 `system.device_info` 相关IP信息就会被覆盖，切记！
