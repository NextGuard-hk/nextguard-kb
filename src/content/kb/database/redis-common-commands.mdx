## Redis常用命令

### 1、查眎key类型

```bash
redis 127.0.0.1:6379[1]> type  network_incident_id_send_set_key
```

**1、zset**

查看方式：zcard

```bash
redis 127.0.0.1:6379[1]> zcard  network_incident_id_send_set_key  #查看有多少数据，里面记录发送成功的事件id
(integer) 56379
```

**2、hash**

查看方式：hgetall

```bash
redis 127.0.0.1:6379[1]> type network_incident_id_list_key_temp_map
hash
查看内容：
redis 127.0.0.1:6379[1]> hgetall network_incident_id_list_key_temp_map
1) "100"
2) "32"
```

**3、string**

查看方式：get

```bash
127.0.0.1:6379> get "backup-20231217010009-d622eaad-9157-4eaa-b9f1-1d84777698b4"
```

### 2、key类型

**zset是个有序且不可重复的数据集合**

```bash
redis 127.0.0.1:6379[1]> type  network_incident_id_send_set_key
zset
redis 127.0.0.1:6379[1]> zcard  network_incident_id_send_set_key  #查看有多少数据，里面记录发送成功的事件id
(integer) 56379
127.0.0.1:6379[1]> zadd network_incident_id_send_set_key 6317620 6317620  #写入新的数据，写入事件id后，系统就不会再次尝试发送邮件
(integer) 0
zRemRangeByScore network_incident_id_send_set_key 事件id 事件id  #移除里面的指定数据，移除后，才能通过命令重新投递。
zrange network_incident_id_send_set_key 0 -1 withscores  #查看数据，最大缓存20W条
```

```bash
hset network_incident_id_list_key_temp_map 事件id 0  #里面包含发送失败的事件id，需要手动hset 0后，才会加入队列
```

- `zRemRangeByScore` #除有序集key中，所有score值介于min和max之间(包括等于min或max)的成员
- 查眎zset类型数据内容：`zcard key`

### 3、设置字段，并写入和读取

```bash
127.0.0.1:6379[1]> set aa 111
OK
127.0.0.1:6379[1]> get aa
"111"
```
