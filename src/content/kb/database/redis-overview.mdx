## 03.REDIS数据库

3.16改为：

```bash
sgcache-cli  -n 1 -a Redis@SkyGuard253.2
sgcache-cli  -n 6379 -a Redis@SkyGuard253.2
```

入库es一次处理10条信息（json），最大支持10个线程，最多100个json（一个json可能包含多个事件），Hotfix-SPS-3.3.0-HF103_amd64.deb开始已经优化为一次处理20个json了

### 事件：

1、事件存在redis，事件先到redis的待分拣区，然后redis按各种索引分类进行存放写入表单

2、sps从redis中取出临时事件，并记录在sps-incident.log中，并最终写入到ES

备注：reidis的自增id就是是网络事件文件的索引，如果自增id不连续，sps就拿不到对应的事件了，会导致sps-incident不会触发通知邮件，但是索引丢了，事件入库没问题。

### 用户行为日志/邮件日志：

这些文件虽然通过sps-transfer传递到ucss的临时目录，但是uuid、用户目录、分级信息需要缓存到redis，不然每次入库都需要从pg数据库查询组织关系会很慢。

### 案例2：数据库查眎AD增量还是全量同步

```bash
进入数据库：redis-cli -n 1
HGETALL "ldap_full_synchronize"  #查看同步记录 true表示全量同步
```

PG数据库查看页面配置生效的是增量还是全量：

```sql
select * from system.sys_props;
```

echo 红色区域 |base64 -d，MD5解码后，可确认当前页面生效的是全量还是增量

如果ad同步异常，导致页面一直处于正在同步字段，可以把key为：ldap_is_synchronizing的值由true改为false即可

### 判断邮件是内部域还是外部域

APP-024之后，升级了Redis，卸载的时候我们考虑了清理掉高版本的rdb文件（格式不兼容），但是有时候在删除之后，又遇到Redis的回写，文件又写出来了。

造成的现象是，redis反复重启

处理办法：Host里删除掉 `/var/skyguard/redis/data/redis.rdb`
