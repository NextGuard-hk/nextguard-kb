# Quick Start Guide

This guide walks you through deploying Nextguard and protecting your first endpoint in under 30 minutes.

## Prerequisites

Before you begin, ensure you have:

- A valid Nextguard license key
- Network access to install the UCSS appliance
- Administrator credentials for your environment
- Supported browser: Chrome 90+, Firefox 88+, Edge 90+

## Step 1: Access the Management Console

1. Open your browser and navigate to `https://<UCSS-IP>:8443`
2. Accept the SSL certificate warning (or install the Nextguard CA cert)
3. Log in with:
   - **Username:** `admin`
   - **Password:** (provided in your welcome email)
4. You will be prompted to change your password on first login

## Step 2: Upload Your License

1. Navigate to **System > License**
2. Click **Upload License**
3. Select your `.lic` license file
4. Click **Activate**
5. Verify that the license shows:
   - Status: **Active**
   - Expiry date is correct
   - Feature set matches your subscription

## Step 3: Configure Network Settings

```
System > Network > Management Interface
```

| Setting | Recommended Value |
|---------|-------------------|
| IP Address | Static IP in your management VLAN |
| Subnet Mask | /24 or as per your network |
| Default Gateway | Your network gateway |
| DNS Server | Your internal DNS or 8.8.8.8 |
| NTP Server | pool.ntp.org or internal NTP |

## Step 4: Deploy Your First Endpoint Agent

### Download the Agent

1. Go to **Devices > Agent Downloads**
2. Select your platform:
   - **Windows** (MSI installer, 64-bit)
   - **macOS** (PKG installer)
   - **Linux** (DEB/RPM package)
3. Download the installer

### Install on Windows

```powershell
# Silent installation
msiexec /i NextguardAgent_x64.msi UCSS_SERVER=https://<UCSS-IP>:8443 /quiet
```

### Install on macOS

```bash
sudo installer -pkg NextguardAgent.pkg -target /
```

### Install on Linux (Ubuntu/Debian)

```bash
sudo dpkg -i nextguard-agent_amd64.deb
sudo systemctl enable nextguard-agent
sudo systemctl start nextguard-agent
```

## Step 5: Verify Agent Registration

1. Go to **Devices > All Devices** in the UCSS console
2. Your new device should appear within 2-3 minutes
3. Status should show **Online** (green dot)
4. Click the device to verify:
   - Agent version is current
   - Last seen time is recent
   - Policies are applied

## Step 6: Apply Your First DLP Policy

1. Navigate to **Policies > DLP Policies**
2. Click **Create Policy**
3. Select a template:
   - **Financial Data Protection** — blocks credit card, bank account numbers
   - **Personal Data (GDPR)** — protects PII across endpoints
   - **Intellectual Property** — prevents source code leakage
4. Configure actions:
   - **Block** — prevent the transfer
   - **Warn** — alert user but allow
   - **Log Only** — silent monitoring
5. Assign to device groups or individual devices
6. Click **Save & Deploy**

## Step 7: Test the Policy

1. On the enrolled device, attempt to copy sensitive data (e.g., a test credit card number like `4111 1111 1111 1111`) to an external drive or email
2. Verify the policy triggers:
   - **Block**: User sees a Nextguard notification
   - **Warn**: User receives a warning prompt
3. Check the event in **Reports > DLP Events**

## You're Protected!

Congratulations — your first Nextguard DLP policy is active.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Device not appearing | Check firewall allows TCP 8443 from device to UCSS |
| Policy not applying | Verify device group assignment and policy priority |
| Agent won't install | Run installer as Administrator / with `sudo` |
| License upload fails | Confirm the serial number matches System > System Info |
