# Logs & Reporting

NextGuard Email Security Gateway provides comprehensive logging and reporting capabilities for security monitoring, compliance auditing, and operational visibility. This guide covers how to access logs, configure log settings, and generate reports.

---

## Log Types Overview

| Log Type | Description | Retention Default |
|----------|-------------|-------------------|
| Mail Log | All inbound/outbound SMTP transactions | 90 days |
| Spam Log | Messages flagged or blocked by anti-spam | 90 days |
| Virus Log | Malware and virus detections | 180 days |
| Policy Log | Content filter, DLP, and policy rule hits | 90 days |
| Anti-Spoofing Log | SPF/DKIM/DMARC check results | 90 days |
| Quarantine Log | Quarantine actions (hold, release, delete) | 90 days |
| Admin Audit Log | Admin console login and configuration changes | 365 days |
| System Log | Gateway service events, errors, health status | 30 days |

## Accessing Logs

### Mail Log (Email Activity Log)

The mail log provides a complete record of all email transactions.

1. Navigate to **Logs > Mail Log**
2. Use filters to search:
   - **Date range:** Select start and end date/time
   - **Direction:** All / Inbound / Outbound
   - **Sender:** Email address or domain
   - **Recipient:** Email address
   - **Subject:** Keyword search
   - **Status:** Delivered, Quarantined, Rejected, Deferred, Bounced
   - **Size:** Filter by message size range
3. Click **Search**

### Log Entry Details

Click any log entry to view full details:

| Field | Description |
|-------|-------------|
| Timestamp | UTC timestamp of SMTP transaction |
| Message ID | RFC 2822 Message-ID header |
| Envelope From | SMTP `MAIL FROM` address |
| Envelope To | SMTP `RCPT TO` address |
| Header From | Display `From:` header |
| Subject | Email subject line |
| Size | Total message size (bytes) |
| Direction | Inbound / Outbound |
| Status | Final delivery status |
| Spam Score | Composite anti-spam score |
| Policy Action | Which policy was triggered |
| Gateway IP | Processing gateway IP address |
| Sender IP | Originating MTA IP address |

## Spam and Threat Logs

### Anti-Spam Log

1. Navigate to **Logs > Spam Log**
2. View all messages with spam score ≥ threshold
3. Filter by:
   - Spam score range
   - Action taken (tagged, quarantined, rejected)
   - Sender domain
   - Date range

### Anti-Virus Log

1. Navigate to **Logs > Virus Log**
2. View all virus/malware detections
3. Each entry includes:
   - Virus/malware name and signature ID
   - File name and MIME type
   - Action taken (quarantined/rejected)
   - Sandbox analysis result (if applicable)

### Anti-Spoofing Log

1. Navigate to **Logs > Anti-Spoofing Log**
2. View SPF, DKIM, and DMARC check results
3. Filter by:
   - Check type (SPF / DKIM / DMARC)
   - Result (Pass / Fail / SoftFail / None)
   - Sender domain

## Admin Audit Log

The admin audit log records all administrative actions for accountability.

1. Navigate to **Logs > Admin Audit Log**
2. View entries for:
   - Admin login and logout events
   - Policy configuration changes
   - Allow/block list modifications
   - Quarantine release/delete actions
   - System setting changes
   - User account modifications
3. Each entry includes: timestamp, admin username, action type, affected object, before/after values

> **Compliance:** Admin audit logs should be exported monthly for ISMS, ISO 27001, or regulatory compliance audits.

## Log Export and Forwarding

### Exporting Logs to CSV/PDF

1. Navigate to the desired log section
2. Apply filters for the desired date range and criteria
3. Click **Export**
4. Select format:
   - **CSV:** For spreadsheet analysis
   - **PDF:** For compliance reporting
5. Click **Download**

### Syslog Forwarding (SIEM Integration)

Forward logs to a SIEM or syslog server in real time:

1. Navigate to **Settings > Log Forwarding > Syslog**
2. Click **Add Syslog Server**
3. Configure:
   - **Server IP:** e.g., `192.168.1.200`
   - **Port:** 514 (UDP) or 6514 (TCP/TLS)
   - **Protocol:** UDP / TCP / TCP with TLS
   - **Format:** RFC 3164 (BSD) or RFC 5424 (IETF)
   - **Log types to forward:** Select from Mail, Spam, Virus, Policy, Admin
4. Click **Test Connection**
5. Click **Save**

### Supported SIEM Integrations

| SIEM Platform | Integration Method |
|--------------|-------------------|
| Splunk | Syslog or Splunk HEC |
| IBM QRadar | Syslog (RFC 5424) |
| Microsoft Sentinel | Syslog via Log Analytics Agent |
| Elastic SIEM | Filebeat syslog input |
| Generic SIEM | Syslog UDP/TCP |

## Built-In Reports

### Email Traffic Summary Report

1. Navigate to **Reports > Email Traffic**
2. Configure:
   - **Date range:** Daily, Weekly, Monthly, Custom
   - **Direction:** Inbound / Outbound / Both
3. Report includes:
   - Total messages processed
   - Delivered vs. blocked ratio
   - Top senders and recipients by volume
   - Bandwidth usage (MB)
   - Peak traffic hours (hourly distribution chart)

### Threat Detection Report

1. Navigate to **Reports > Threat Detection**
2. Report includes:
   - Spam detected by day
   - Virus detections by type
   - Anti-spoofing failures
   - Phishing / BEC attempts blocked
   - Sandbox detonation results
   - Blocked attachment types

### Anti-Spam Report

1. Navigate to **Reports > Anti-Spam**
2. Report includes:
   - Total spam blocked
   - Spam score distribution histogram
   - Top spam senders by domain
   - RBL match breakdown
   - False positive rate (if user-reported)
   - Quarantine release rate

### Anti-Virus Report

1. Navigate to **Reports > Anti-Virus**
2. Report includes:
   - Total viruses detected
   - Detection by engine (signature vs. sandbox)
   - Top malware families
   - Blocked file types
   - Trend chart over time

### User Activity Report

1. Navigate to **Reports > User Activity**
2. Report includes:
   - Email volume per user (sent/received)
   - Policy violations per user
   - Quarantine release actions per user
   - Users with most spam received

## Scheduled Reports

Configure automatic report delivery:

1. Navigate to **Reports > Scheduled Reports**
2. Click **Add Schedule**
3. Configure:
   - **Report type:** Select from available reports
   - **Frequency:** Daily / Weekly / Monthly
   - **Delivery time:** e.g., 07:00 Monday
   - **Recipients:** Email addresses of report recipients
   - **Format:** PDF or CSV attachment
4. Click **Save**

### Recommended Scheduled Reports

| Report | Frequency | Recipients |
|--------|-----------|------------|
| Email Traffic Summary | Weekly | IT Manager |
| Threat Detection | Daily | Security Team |
| Anti-Spam Summary | Weekly | IT Admin |
| Admin Audit Log | Monthly | CISO / Compliance |
| Quarantine Summary | Weekly | IT Helpdesk |

## Log Retention Configuration

1. Navigate to **Settings > Log Retention**
2. Set retention periods per log type:
   - **Mail Log:** 90 days minimum (recommended: 180 days)
   - **Threat Logs:** 180 days minimum
   - **Admin Audit Log:** 365 days (compliance requirement)
3. Enable **Log Archiving** to external storage:
   - Configure NFS/CIFS mount or S3-compatible object storage
   - Archived logs are compressed and indexed
4. Click **Save**

> **Regulatory note:** PCI-DSS requires 12 months of audit log retention. HIPAA requires 6 years. Configure accordingly.

## Dashboard Overview

The Email Gateway Dashboard provides real-time metrics:

1. Navigate to **Dashboard > Email Gateway**
2. View real-time panels:
   - **Messages/hour:** Live traffic graph
   - **Threat summary:** Spam, virus, spoofing counts (last 24h)
   - **Quarantine queue size:** Current held messages
   - **System health:** CPU, memory, disk usage
   - **Top threat sources:** Map of source IPs
   - **Recent incidents:** Latest high-severity detections
