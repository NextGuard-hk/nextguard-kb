# Quarantine & Mail Trace

NextGuard Email Security Gateway provides a centralized **quarantine** for holding suspicious or policy-violating emails, and a **mail trace** tool for tracking message delivery status. This guide covers quarantine management, end-user release workflows, and how to use mail trace for troubleshooting.

---

## Quarantine Overview

The quarantine stores emails that have been flagged by the following policies:

| Policy | Quarantine Trigger |
|--------|--------------------|
| Anti-Spam | Spam score ≥ configured quarantine threshold |
| Anti-Virus | Malware or virus detected in attachment |
| Anti-Spoofing | SPF/DKIM/DMARC failure (if policy = quarantine) |
| Content Filter | Keyword, attachment type, or data classification match |
| Sandbox | Suspicious file pending sandbox analysis |

---

## Accessing the Quarantine Queue

### Admin Quarantine View

1. Log in to the **NextGuard Admin Console**
2. Navigate to **Email Gateway > Quarantine**
3. View all quarantined messages across all users

### Filtering Quarantine

Use the filter bar to narrow results:

| Filter | Options |
|--------|--------|
| Date range | Today, Last 7 days, Last 30 days, Custom |
| Quarantine reason | Spam, Virus, Spoofing, Content Filter, Sandbox |
| Sender | Filter by sender email or domain |
| Recipient | Filter by recipient email |
| Subject | Keyword search |
| Status | Pending, Released, Deleted, Expired |

## Managing Quarantined Emails

### Viewing a Quarantined Message

1. Click on a message in the quarantine list
2. View:
   - **Message headers:** Full SMTP headers
   - **Body preview:** Safe HTML rendering of email body
   - **Attachments:** Listed with hash values (do not open directly)
   - **Quarantine reason:** Detailed policy trigger information
   - **Spam score breakdown:** Per-layer scoring details

### Releasing a Quarantined Email

To deliver a quarantined email to the original recipient:

1. Select the message(s) in the quarantine list
2. Click **Release**
3. Choose release option:
   - **Release only:** Deliver to recipient
   - **Release and allow sender:** Add sender to allow list
   - **Release and train:** Mark as false positive for Bayesian training
4. Confirm the action

> **Note:** Releasing a virus-infected email is blocked by default. Admin must explicitly override virus quarantine with elevated permissions.

### Deleting Quarantined Emails

1. Select message(s)
2. Click **Delete**
3. Confirm deletion

Messages are permanently deleted and cannot be recovered.

### Bulk Actions

1. Check **Select All** to select all filtered results
2. Choose bulk action:
   - **Release All Selected**
   - **Delete All Selected**
   - **Export to CSV** (for audit/compliance)

## Quarantine Retention Settings

Configure how long messages are retained in quarantine:

1. Navigate to **Email Gateway > Settings > Quarantine Retention**
2. Set retention periods by category:
   - **Spam:** 14 days (default)
   - **Virus:** 30 days (default)
   - **Content Filter:** 30 days (default)
   - **Sandbox:** 7 days (default)
3. Enable **Auto-delete expired messages**
4. Click **Save**

> **Compliance note:** For regulated industries, consider extending retention to 90 days for audit purposes.

## End-User Quarantine Self-Service

### Enabling End-User Access

1. Navigate to **Policy > Anti-Spam > End-User Settings**
2. Enable **User Self-Service Quarantine**
3. Configure permissions:
   - **View own quarantine:** Yes
   - **Release spam:** Yes
   - **Release virus:** No (admin only)
   - **Delete messages:** Yes
   - **Add to personal allow list:** Yes
4. Click **Save**

### User Self-Service Portal

End users access their personal quarantine at:
```
https://<gateway-hostname>/user-portal
```

Users can:
- View emails quarantined for their address
- Release or delete individual messages
- View spam score details
- Manage their personal allow/block lists

### Quarantine Digest Email

Automatic digest notifications summarize quarantined emails:

1. Navigate to **Policy > Anti-Spam > Quarantine Digest**
2. Configure:
   - **Frequency:** Daily or Twice daily
   - **Delivery time:** 08:00
   - **Sender address:** `quarantine@yourdomain.com`
   - **Include release links:** Yes (one-click release)
3. Click **Save**

Sample digest includes:
- Date/time quarantined
- Sender email address
- Subject line
- Spam score
- Release / Delete / Preview links

## Mail Trace

Mail Trace tracks the delivery status of all inbound and outbound email processed by the NextGuard gateway.

### Accessing Mail Trace

1. Navigate to **Email Gateway > Mail Trace**
2. Enter search criteria:
   - **Sender:** `sender@example.com` or domain `@example.com`
   - **Recipient:** `user@yourdomain.com`
   - **Subject:** Partial keyword match
   - **Message ID:** Exact SMTP Message-ID header value
   - **Date range:** From / To date and time

### Mail Trace Results

Each result row shows:

| Column | Description |
|--------|-------------|
| Timestamp | Date and time the message was processed |
| Direction | Inbound / Outbound |
| Sender | Envelope SMTP sender (`MAIL FROM`) |
| Recipient | Envelope SMTP recipient (`RCPT TO`) |
| Subject | Email subject line |
| Size | Message size in KB/MB |
| Status | Delivered, Quarantined, Rejected, Deferred, Bounced |
| Policy | Which policy triggered action (if any) |
| Spam Score | Composite spam score |

### Viewing Full Message Trace Details

Click on a message to view:

1. **SMTP transaction log:** Full SMTP conversation
2. **Policy evaluation:** Step-by-step results from each security layer
3. **Delivery attempt log:** MTA delivery attempts with timestamps
4. **Message headers:** Complete RFC 2822 headers
5. **Quarantine status:** If quarantined, shows quarantine queue entry

### Common Mail Trace Use Cases

**Email not received by user:**
1. Open Mail Trace
2. Search by recipient email and approximate date/time
3. Check **Status** column:
   - **Quarantined:** Release from quarantine
   - **Rejected:** Review rejection reason (spam/virus/policy)
   - **Deferred:** Check external MX DNS or downstream mail server
   - **Delivered:** Confirm delivery to internal mail server

**Email sent but not received externally:**
1. Open Mail Trace
2. Set direction to **Outbound**
3. Search by sender and recipient
4. Check delivery status and bounce reasons

**Investigating a phishing report:**
1. Search by sender domain
2. Check how many recipients received the email
3. Identify policy hits (spam score, spoofing checks)
4. If delivered: manually quarantine and notify recipients

## Quarantine Notifications for Admins

### Admin Alert Settings

1. Navigate to **Policy > Notifications > Quarantine Alerts**
2. Configure alerts for:
   - **New virus detected:** Immediate alert to security team
   - **High-volume spam campaign:** Alert when > 100 messages/hour quarantined
   - **Quarantine capacity warning:** Alert when quarantine > 80% full
3. Set alert recipients: `security-team@yourdomain.com`
4. Click **Save**

## Quarantine Reports

### Generating Quarantine Summary Reports

1. Navigate to **Reports > Quarantine Summary**
2. Set date range and report parameters
3. View metrics:
   - Total messages quarantined
   - Breakdown by quarantine reason
   - Top senders of quarantined email
   - Release rate (percentage of quarantined emails released)
   - False positive rate

### Exporting Quarantine Data

1. Navigate to **Reports > Quarantine Summary**
2. Click **Export**
3. Select format: CSV or PDF
4. Choose date range
5. Click **Download**

> **Compliance:** Export quarantine logs monthly for audit trail documentation.

## Best Practices

| Recommendation | Detail |
|---------------|--------|
| Review quarantine daily | Catch false positives before retention expiry |
| Send quarantine digests to users | Reduces IT helpdesk load for "missing email" tickets |
| Enable admin alerts for new virus detections | Enables rapid incident response |
| Set 30-day retention for compliance | Supports audit requirements |
| Disable virus release for end users | Prevents accidental malware delivery |
| Use mail trace before escalating | Most "email not received" issues resolved in seconds |
