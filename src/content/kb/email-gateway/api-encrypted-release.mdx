## 概述

当邮件被SEG隔离后，管理员可以通过API接口对隔离邮件进行加密释放投递。本文档记录了使用QKAct API进行加密投递的完整流程。

## 查看隔离邮件

在SEG管理界面中，进入 **监控 > 邮件监控 > 邮件日志** 查看被隔离的邮件：

- 可以按接收时间筛选（如最近3天）
- 查看邮件ID、SMTP-ID、邮件方向等信息
- 被隔离的邮件会显示"命中"标记

## API加密释放

### 接口信息

- **接口路径：** `/qkact/v0/seg/encrypt/delivery`
- **请求方法：** POST
- **协议：** HTTPS（端口8443）

### 请求参数

```json
{
  "smtpId": {
    "value": "<SMTP-ID值>"
  },
  "encrypt": true
}
```

| 参数 | 说明 |
|---|---|
| `smtpId.value` | 邮件的SMTP-ID，可从邮件日志中获取 |
| `encrypt` | 设置为 `true` 表示加密投递 |

### 响应结果

成功响应：

```json
{
  "status": 0,
  "reason": "成功加入投递队列"
}
```

## 日志验证

在DSG-MTA的日志中可以验证加密投递是否成功：

```
encrypt to recipients ok  # 表示加密投递成功
```

### 关键日志信息

投递过程中，DSG-MTA的 `postfix/policy` 日志会显示以下关键步骤：

1. **策略匹配：** `policy socket` 相关日志
2. **附件处理：** `attachment` 匹配和 `match_list` 检查
3. **加密执行：** `aes256 algorithm`、`encrypt to recipients ok`
4. **投递完成：** `send attr` 和 `Queued` 状态确认

> **注意：** 加密释放后，密钥邮件会通过MTA发送给收件人，确保MTA到收件人域名的路由可达。
