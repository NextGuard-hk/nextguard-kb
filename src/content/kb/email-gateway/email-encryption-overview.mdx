## 修改加密外发通知模板

当邮件DLP策略执行为邮件加密动作时，命中策略后，邮件密钥邮件会发送给收件人。

### 模板文件位置

模板所在目录是持久化的，文件位置是：

```
/etc/ats/postfix/notify_zh.cf
```

在中文模板上修改，修改之前备份，改完执行 `postfix reload` 生效。

### 模板内容结构

模板中使用以下宏变量：

| 宏变量 | 说明 |
|---|---|
| `MACROMESSAGEID` | 邮件Message-ID |
| `MACROFROM` | 发件人地址 |
| `MACROSUBJECT` | 邮件主题 |
| `MACROTO` | 收件人地址 |
| `MACROKEY` | 解密秘钥 |
| `MACROORISUBJECT` | 原始邮件主题 |
| `MACROORISENDER` | 原始发件人 |
| `MACROORIRECVERS` | 原始收件人 |

## 主题修改

发件主题修改：

> **注意：** 发件主题只支持修改前缀，也就是“邮件已被加密”这部分，后缀是原有主题不支持修改。

修改方法：

```bash
dcin ats
postconf -e notify_subject_prefix=自定义前缀  # 把“自定义前缀”换成客户要定义的内容
postfix reload  # 生效配置
```

## 邮件加密外发注意事项

### 1. 密钥邮件发送路由

密钥给发件人地址，是通过 **MTA** 口去发送的，需要保证MTA到发件域名路由可达。

SEG的高级路由需要配置邮件服务器的IP地址和域名，如果不配置，SEG默认使用DNS解析，可能无法把密钥投递给内网发件人域名。

### 2. 加密方式配置

原文加密还是附件加密、密钥发给发件人还是收件人、加密方式：取决于 **DSG高级配置** 中的加密选项。

### 3. DLP策略动作区别

DLP策略动作中，“邮件内容加密”和“隔离-加密释放”有区别：

1. **直接加密投递：** DLP选择内容加密后，DSG/MTA高级配置选择内容/附件加密，邮件经过MTA由MTA直接完成内容/附件加密。
2. **隔离后加密释放：** DLP选择隔离并释放加密后，SPS先生成网络事件，主管审批释放后，按照流程把证据送还MTA时会有特殊标志（加密投递），MTA识别特殊标志进行内容加密投递。

### 4. 加密外发两种情况

1. 命中策略后，直接加密投递出去，通知由 **DSG-MTA** 发送。
2. 命中策略后，先隔离，审批释放后加密投递。

### 5. 日志排查

在 `mail.log` 日志中可以查看加密外发的打印信息，例如：

```
2024-03-01T11:37:32.447643+08:00 DSG-MTA postfix/smtp[51327]: to=<user@example.com>, status=sent
```
