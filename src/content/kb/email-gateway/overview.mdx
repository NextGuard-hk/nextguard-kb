## What is the Email Security Gateway?

The Nextguard **Email Security Gateway (ESG)**, also known as **SEG (Secure Email Gateway)**, is an enterprise email security appliance that protects your organization from email-borne threats and prevents sensitive data from leaving via email.

It sits in the email flow path, inspecting all inbound and outbound email in real time.

---

## Architecture Overview

```
[Internet / External Senders]
           |
    [DNS MX Record]
           |
  [Nextguard ESG / SEG]
    - Anti-Spam
    - Anti-Virus
    - Anti-Spoofing
    - Email DLP
    - Encryption
           |
  [Internal Mail Server]
  (Exchange / Office 365)
```

The ESG operates as an **MTA (Mail Transfer Agent)** — your MX record points to the ESG, which processes and forwards email to your internal mail server.

## Core Capabilities

### Inbound Protection
| Feature | Description |
|---------|-------------|
| **Anti-Spam** | Multi-layer spam detection and filtering |
| **Anti-Virus** | Real-time scanning of email and attachments |
| **Anti-Spoofing** | SPF, DKIM, DMARC enforcement |
| **URL Rewriting** | Rewrites links to protect against phishing |
| **Attachment Sandboxing** | Dynamic analysis of suspicious files |
| **Header Analysis** | Detects forged sender headers |

### Outbound Protection
| Feature | Description |
|---------|-------------|
| **Email DLP** | Scans outbound content for sensitive data |
| **Encryption** | Automatic TLS/PGP/S/MIME encryption |
| **DKIM Signing** | Signs outbound email for authentication |
| **SPF Alignment** | Ensures outbound mail passes SPF checks |

### Management
| Feature | Description |
|---------|-------------|
| **Quarantine** | Holds suspicious email for admin/user review |
| **Mail Trace** | Full email tracking and delivery status |
| **Reports** | Spam statistics, virus reports, DLP incidents |
| **UCSS Integration** | Centrally managed via UCSS platform |

---

## Email Flow

**Inbound Email Flow:**
1. External sender sends email
2. MX DNS record routes to ESG
3. ESG checks: SPF/DKIM/DMARC, reputation, spam score, AV scan
4. Clean email forwarded to internal mail server
5. Suspicious email quarantined or tagged

**Outbound Email Flow:**
1. Internal user sends email
2. Mail client/server routes through ESG
3. ESG applies: DLP scan, DKIM signing, TLS enforcement
4. Email delivered to external recipient

## Supported Mail Environments

| Environment | Integration Method |
|-------------|-------------------|
| Microsoft Exchange | SMTP relay (smart host) |
| Microsoft 365 (Exchange Online) | Connector + MX record |
| Google Workspace | SMTP relay |
| Other SMTP servers | MTA relay configuration |

---

## Deployment Requirements

| Item | Requirement |
|------|-------------|
| MX Record | Must point to ESG public IP |
| Outbound Smart Host | Internal mail server routes outbound via ESG |
| DNS | ESG must resolve external MX/A records |
| Ports | 25 (inbound SMTP), 587 (submission), 465 (SMTPS) |
| Management | UCSS accessible from ESG |
