# KB-ES Proxy API日志查询使用手册

## 1. 认证方式 (Check-in Check-out)

### 概述

在客户端访问UCSS API接口前，客户端必须向UCSS申请令牌。令牌申请需要在请求header中包含正确的用户名和密码，其中密码需要密文传送，使用3DES加密方式进行加密，加密的密钥需登录ucss管理平台在管理员页面申请access-key。

验证通过后可成功申请令牌。客户端在后续请求中需将令牌返回给客户端。令牌有过期时间限制，使用过期的令牌会导致UCSS返回“无效令牌”错误。

## 2. 密码加密使用方式

找到API账号点击进入编辑页面（测试环境使用admin登录使用API）。

## 3. API接口说明

### Check-in接口

- **URL**: `https://UCSS_URL/qkact/v0/si/check-in`
- **Method**: POST
- **Header**: 
  - Content-Type: application/json
  - Authorization: Basic base64(user:encrypted_password)

### 日志查询接口

- **URL**: `https://UCSS_URL/qkact/v0/es-proxy/log-query`
- **Method**: POST
- **Header**:
  - Content-Type: application/json
  - Authorization: Bearer {token}

### Check-out接口

- **URL**: `https://UCSS_URL/qkact/v0/si/check-out`
- **Method**: POST

## 4. 注意事项

1. 令牌有时效性，过期后需重新申请
2. 密码传输必须使用3DES加密
3. 查询完成后应调用check-out释放令牌
