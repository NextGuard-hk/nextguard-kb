# 页面收集日志失败问题汇总

## 1. ucss管理日志收集超时

### 原因

定位时由于sps_client.py脚本中请求超时时间过短（默认12秒），在网络环境不好时容易超时。

### 解决方法

修改sps_client.py中的timeout参数：

```python
def set_timeout(self, timeout=12):
    socket.setdefaulttimeout(timeout)
```

可以将timeout值调大。

## 2. 系统运行日志收集失败

### 原因

客户环境中无法收集UCSS日志的根本是由于后台日志文件不存在。

### 解决方法

手工创建对应的文件后，系统日志收集恢复正常：

```bash
cd /var/log/skyguard/app
touch lcserver.log.202303020000
```
