# KB6-导入授权时，提示"推送license到后台Appliance失败"问题汇总

## 场景1：容器化版本UCSS设备导入授权时，提示"推送license到后台Appliance失败"

**分析：**
查看manager.log日志，发现报错信息。进入spe容器，发现spe-service、dlpserver无法启动。DMServer.xml配置文件大小为0kb。

查找0kb的文件：
```
find /opt/skyguard/ucs/etc/ -size 0
```

Appliance会把SPS推送来的License再推给后台各组件，如果license到后台失败，可能是DLPServer.xml、DMServer.xml配置文件为0kb的问题。

在初始化系统时，会有较大量的写入操作，操作系统会先缓存到内存，丢到写入队列（类似于我们写U盘一样，丢一个文件到U盘上，会看到Sync数据）。如果往硬盘上Sync的操作没有完成，非正常关机或掉电，因为文件系统要保证一致性，所以只要写入操作没有最终完成，要么文件内容截断或者干脆丢失。严重的文件系统不一致，会导致启动中断。

**解决办法：**
1. 新项目建议重新安装，由于无法快速清点影响的文件数量。关机、掉电、磁盘空间满才会有这种情况，需要具体评估
2. 虚拟化平台安装，如果是直接克隆方式部署，请遵循以下步骤：
   - a. 创建虚拟机，导入镜像安装母版系统
   - b. 安装完成后，系统会自动重启，并进入选择设备（"Server"），等等待系统完全关闭，再生成模板或者直接克隆
   - c. 不要克隆正在运行的虚拟机，一定要遵循步骤b关机后再克隆
   - d. 系统初始化时会有较大量的写入操作，请不要在初始化期间掉电关机
3. 对于运行时的，非持久化目录中文件的损坏（非系统盘），可清除缓存解决

## 场景2：容器化版本的ASWG/DSG设备导入授权时，提示"推送license到后台Appliance失败"

**分析：**
出现此问题一般是ASWG/DSG的Appliance组件推送注册信息失败。进入ASWG后台，docker ps，发现ats容器启动时间较短。由于apache2组件就在ats容器中，所有注册失败。

**解决办法：**
给ASWG/DSG配置4张网卡。配置4张网卡，缺一不可。

## 场景3：策略配置迁移时，还原配置到新的UCSS服务器后，进行初始化授权失败

**分析：**
导入的配置文件中包含了之前设备的deviceid、token信息不一致。

**解决办法：**
初始化UCSS即可（UCSS初始化的过程，也叫自身信息注册）

## 场景4：UCSS导入授权时报错，推送License到后台Appliance失败

**分析：**
排除其他问题，可能是sps组件和Appliance组件通信异常。

**解决办法：**
重启apache2和sps-webservice服务即可解决

## 场景5：ASWG等网关设备分别先后在UCSS1、UCSS2上注册过，导入授权失败

**分析：**
ASWG等网关设备Appliance中记录的授权token信息与UCSS中的不一致。

**解决办法：**
重新注册到ucss，再次导入license文件即可。

## 场景5续：ASWG等网关设备导入授权时，提示"推送license到后台Appliance失败"

**分析：**
初始化后重新注册到ucss，会提示注册失败。导入的license文件不完整，导致ucss认证不通过。

**解决办法：**
删除/opt/skyguard/cert/license_sig授权文件后重试

## 场景6：客户环境防火墙出向放行了license.skyguard.com.cn

**原因分析：**
客户环境防火墙出向放行了license.skyguard.com.cn数据导致代码一直在那里等响应。

备注：授权文件是支持离线导入，例如如果和license服务器断开也可以导入。
manager.log报错：Can't connect to license server

**解决办法：**
1. 防护墙加白license.skyguard.com
2. 或者hosts修改ucss和seg的127.0.0.1

## 场景7：查看下述日志，均没有收到sps的推送日志

检查日志文件：
```
/var/log/skyguard/app/app.log
/var/log/skyguard/app/appagent.log
/var/log/nginx/access.log
```

查看sps的manager.log日志，截图如下，提示异常。

**原因：清除浏览器缓存**

## 场景8：客户3.10的centos环境，打app-010打不上

**分析过程：**
1. 导入license过程在，app.log无相关打印信息
2. 删掉/opt/skyguard/cert/license_sig授权文件

**原因：**
客户原先打app-010未打成功，导致/opt/skyguard/相关路径异常

**解决办法：**
1. 后台卸载未安装成功的app-010
   /opt/skyguard/download/hotfix/workdir/APP-010/uninstall.sh
2. 取消/opt/skyguard/www/app/handler/license相关链接

完成上述两部后，可以正常完成注册。
如果是uos系统，安装app-011之前，还需要打python-beautifulsoup补丁。
