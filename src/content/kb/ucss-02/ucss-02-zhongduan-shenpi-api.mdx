# API-终端审批接口(DLP)

## 接口描述

终端命中事件仅根据管理员设定的策略动作，进行允许或传输。希望通过违规用户的主管或审批员进入管理流程，实现终端用户的主管审批文档审批请求。

## 用户场景

市场办公人员使用邮件客户端向合作伙伴发送市场宣传文件，通过企业内部OA系统发起审批需求，等待财务总监审批后发送。

## 前置条件 (Prerequisite)

采用快速创建策略的认证方式，参考SI check-in，获取有一定时效性的临时令牌。后续需要重新申请令牌。

## 接口说明

### 【V3.9.0】终端状态监控外部接口

- **Request Method**: POST
- **Request URL**: `https://UCSS_URL/qkact/v0/dlp/incident/approval`
- **Content-Type**: application/json
- **Authorization**: Basic base64("client-id:access-key")

### 请求参数

| 参数 | 说明 |
|---|---|
| approver | 审批人（可为邮箱） |
| submitter_name | 提交人 |
| submitter_ip | 提交人IP |
| submitter_email | 提交人邮箱 |
| submitter_fqdn | 提交人FQDN |
| file_size | 文件大小（必须和实际大小一致） |
| file_md5 | 文件的md5值 |
| file_name | 文件名称 |
| channel | 通道，空代表全通道 |
| approved_time | 审批通过时间 |
| expired_time | 审批过期时间 |
| max_num | 最大通过次数，0代表无限次 |
| forensic | 是否记录证据，1-记录，0-不记录 |

### 成功响应

```json
{
  "status": 200,
  "reason": "success"
}
```

> 注意：status 200表示操作成功，-1表示失败。
