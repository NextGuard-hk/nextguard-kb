# 实时黑名单RBL

## RBL概述

RBL（Realtime Blackhole List）即实时黑名单列表，是一个可供查询的IP地址列表，通过DNS查询方式来查找一个IP地址的A记录是否存在，以判断其是否被列入该实时黑名单中。

在该列表中的IP地址表示对外发布过垃圾邮件，因此可以像病毒定义文件一样被垃圾邮件过滤器用于过滤该列表中发出的垃圾邮件。

## RBL工作机制

1. SMTP服务器接受到连接请求
2. 对连接IP进行逆转
3. 对RBL服务器进行查询
4. 如果查询得到结果，则拒绝连接；如果没有得到结果，则继续进行连接

## 配置说明

- **RBL服务器**：默认使用 `rbl.skyguard.cn`
- **多个英文逗号分隔**，最多支持5个
- **动作**：可设置为关闭

## 查询是否被RBL

使用 `nslookup` 查询IP是否在RBL中：

```bash
nslookup <反转IP>.rbl.skyguard.cn
```

- 如果查询结果在 `127.0.0.X` 则还在黑名单中
- 如果不在，则已经移除
- `rbl.skyguard.cn` 查询时需要将IP倒序查询
- `phi.skyguard.cn` 查询域名IP时则不需要

> 一般是开这个，这个是全局RBL。`zen.spamhaus.org` 可以删掉，它主要是检查上一条出口IP，回进下做RBL检查。
