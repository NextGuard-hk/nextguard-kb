# KB-加密外发汇总

## 修改加密外发通知模板

参考EI-10495：修改加密密钥通知邮件模板。

### 需求场景
1. 邮件DLP策略执行为邮件加密动作
2. 命中策略后，邮件密钥邮件发送给收件人
3. 客户需要修改密钥邮件为其他语言

### 模板文件位置

模板所在目录是持久化的，文件位置：
```
/etc/ats/postfix/notify_zh.cf
```

## 主题修改

发件主题只支持修改前缀，默认为"邮件已被加密"。

修改方法：
```bash
dcin ats
postconf -e notify_subject_prefix=自定义前缀
postfix reload  # 生效配置
```

## 发件人修改

通过系统 > 基本设置 > 系统通知中修改发件人邮件地址。

## 邮件加密外发注意事项

### 1. MTA路由配置
密钥给发件人地址是通过MTA口去发送的，需要保证SEG的高级路由中配置了邮件服务器的IP地址。

高级邮件路由需启用，并配置相应的域名和主机名/IP。

### 2. 加密方式选择
- **原文加密**：对邮件原文进行加密
- **附件加密**：仅对附件进行加密

### 3. 密钥发送对象
- 发件人：密钥发给发件人
- 收件人：密钥发给收件人

### 4. 加密类型
- 国密加密
- ZIP加密
