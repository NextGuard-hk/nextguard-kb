# KB-ASEG如何查看标记邮件Hash对应的SMTP-ID

## 背景

在ASEG设备中经常会出现垃圾邮件漏拦的情况，本篇描述的是其中一种情况：垃圾邮件匹配“标记邮件-正常邮件”从而导致漏拦。

## 标记邮件功能说明

开启标记邮件功能后（SEG管理 > 设置 > 检测设置 > 反垃圾 > 标记邮件），可以对邮件日志中保留原文的邮件进行标记，标记有两种类型：正常邮件/垃圾邮件。

其技术原理是通过邮件中的文本内容生成hash值，而后放入到一个hash list中，当需要检查时从上到下顺序匹配，匹配成功后跳出。

### 注意事项

1. 标记邮件是基于内容进行hash，如果邮件被SPF、RBL等连接层面的防护所拦截，请不要使用标记邮件的功能对其进行加白，这样将会混淆标记邮件的hash list
2. 对于大部分内容类似的邮件，理论上标记其中一封即可，不要简单粗暴的全选>标记

## 如何确认邮件命中了标记邮件

将一份邮件标记为垃圾邮件时，ASEG再次收到内容相似的邮件，将会匹配标记邮件hash，在邮件日志>分析结果中也会出现“标记邮件”的字样。

但是当一份邮件匹配了“标记邮件-正常邮件”的情况下，邮件日志将不会有任何提示，只能在后台通过rspam引擎的日志进行查看。

## 查看标记邮件Hash对应的SMTP-ID

通过日志中hash值来找到对应的SMTP-ID，从而判定是哪一封邮件在捷乱。

### 步骤1：登录MySQL

```bash
dcin mysql
mysql -u root -pMysql@SkyGuard253.3
```

### 步骤2：查询hash对应的SMTP-ID

```sql
use segapi;
select * from mail_fuzzy_hashes where hashes like "fbd2f25f%";
```

查询结果会显示smtpID和对应的hashes值，通过smtpID可以在邮件日志中查找到对应的邮件。

### rspam.log日志分析

通过grep命令查看rspam日志：

```bash
grep 65bbe2 /var/log/rspamd/rspamd.log
```

日志中可以看到邮件的处理过程，包括：
- 邮件加载和解析
- 基于内容打分结果
- 标记邮件hash匹配结果
- 最终判定结果

当邮件匹配“标记邮件”的hash值对应用户标记的一封“正常邮件”时，将绕过垃圾邮件的判定。
