# 反向域名解析/RDNS/PTR

反向域名解析(Reverse DNS)用于识别邮件发送方是否来自合法的域。

## 配置项

反向域名解析支持以下检测选项：

| 检测项 | 说明 |
|--------|------|
| 当发件人IP地址的PTR记录不存在时 | 可设置动作 |
| 当发件人IP地址的PTR记录与A记录不匹配时 | 可设置动作 |
| 当发件人IP地址在反向DNS查询过程中发生失败时 | 可设置动作 |
| 当发件人IP地址的PTR记录与SMTP EHLO/HELO的内容不匹配 | 可设置动作 |

## RDNS原理

反向域名解析记录(RDNS)是在供应商DNS服务器上，把申请到的域名的A记录指向邮件服务器MTA的公网IP，这个申请到的域名作为MTA的主机名，也就是EHLO后面的域名。

### 查询方法

```bash
# 查询PTR记录
nslookup -qt=ptr <IP地址>
```

### 备注

- 一般邮件头会记录MTA的出口IP
- 可以通过反查出口公网IP的PTR解析到配置的FQDN
- 也就是要保证出口公网IP的PTR解析要解析到配置的FQDN

### 配置示例

- **FQDN**: `SEG3.15.test-AD.com`
- **启用状态**: 开启
- **全邮件记录**: 启用

### 需要配置的DNS项

以QQ邮件服务器为例：

1. **MX邮件交换记录**：通过 `nslookup -q=mx` 查询
2. **SPF记录**：通过 `nslookup -q=txt` 查询，逐级查出IP地址，建立域名和IP的对应表
3. **RDNS反向域名解析**：通过 `nslookup -q=ptr` 查询
