# UCSS FAQ

## General

**Q: What is UCSS?**
A: UCSS (Unified Content Security System) is the centralized management platform for all Nextguard security appliances.

**Q: Can UCSS manage devices across multiple sites?**
A: Yes. UCSS supports multi-site management with site-specific policies and centralized reporting.

**Q: What browsers are supported?**
A: Chrome 90+, Firefox 88+, Edge 90+, and Safari 14+.

## Deployment

**Q: Can UCSS run as a virtual machine?**
A: Yes. OVA images are available for VMware, Hyper-V, and KVM environments.

**Q: What are the minimum system requirements?**
A: 4 CPU cores, 8GB RAM, 200GB storage. See the Platform Overview for recommended specs.

**Q: Does UCSS support high availability?**
A: Yes. Active-passive HA clustering with automatic failover is supported.

## Management

**Q: How many devices can UCSS manage?**
A: Depends on license tier. Standard supports up to 50 devices, Enterprise supports unlimited.

**Q: Can I manage devices remotely?**
A: Yes. UCSS uses encrypted TLS communication. Devices can be managed across WAN connections.

**Q: Does UCSS support API access?**
A: Yes. RESTful API is available for automation and integration with third-party systems.

## Backup & Recovery

**Q: How do I back up UCSS configuration?**
A: Navigate to System > Backup > Create Backup. Backups include configuration, policies, and user data.

**Q: Can I schedule automatic backups?**
A: Yes. Configure scheduled backups under System > Backup > Schedule.

**Q: How do I restore from backup?**
A: Navigate to System > Backup > Restore, select the backup file, and follow the restoration wizard.
