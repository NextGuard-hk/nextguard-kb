# KB-08 离线环境软件依赖解决办法

## 问题背景

在软件化安装的情况下，我们无法控制操作系统上安装的软件、以及软件版本。在 `install.sh` 的脚本中需要联网下载一些软件的依赖包（主要是 python 的依赖包）。

但是在很多客户的环境中，宿主机无法联网，这种情况下，应该如何进行软件化安装呢？

## 解决思路

找一台可以链接互联网的 Ubuntu 机器，在执行安装脚本之前，收集下系统安装的软件包，在安装成功之后，再收集下系统安装的软件包，对比两者，收集联网下载的软件包名称。

最终通过 `apt` 命令去下载一份到本地，这样在无法联网的机器上手动安装这些软件包后，`install.sh` 也就可以正常的运行过去了。

## 具体操作步骤

### 1. 安装 SkyGuard 之前列出所有已经安装的包

```bash
dpkg -l | awk -F "[ ]+" '{print $2}' > /Before.list
```

### 2. 安装 SkyGuard 之后列出所有已经安装的包

```bash
dpkg -l | awk -F "[ ]+" '{print $2}' > /After.list
```

### 3. diff 命令对比不同，去掉重复项目，输出文件

```bash
diff Before.list After.list | cat | grep '>' | awk '{print $2}' > /PacketDir/Packet.list
```

### 4. 解决 pkgacquire::run (13: 权限不够) 问题

```bash
sudo chown -R _apt:root /PacketDir
```

### 5. 下载 `Packet.list` 的所有软件包到 `/PacketDir` 目录下

```bash
for filename in `cat Packet.list`; do apt-get download $filename; done
```

### 6. 将 PackDir 目录复制到离线的机器上，安装所有软件包

在安装的过程中，一些软件会弹出配置窗口，可以选择随便设置，或者干脆不设置，因为对于一些必要的软件我们的安装脚本和程序会自行修改其中的配置，在安装阶段只要能装上就行。

```bash
dpkg -i *.deb
```

## 补充命令

### apt autoremove

自动移除不需要的依赖包。在安装软件时，apt 会自动帮我们下载软件的依赖包，但是随着用户安装软件的升级或者卸载等操作，这些依赖包不在被已经安装的任何软件所需要，那么它就会出现在 autoremove 的列表中。

### apt --fix-broken install

在使用 apt 安装软件时，它会将软件包先行安装，但是这些软件包因为依赖的关系无法进行 conf 配置，相当于安装了一半，这个时候我们就需要将他们卸载掉。
