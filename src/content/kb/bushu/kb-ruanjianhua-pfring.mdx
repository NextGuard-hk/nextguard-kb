# KB-软件化部署环境手动安装pf_ring方法

## 背景

旁路产品（DSG旁路、SWG旁路）需要用到内核模块，与内核版本、配置、版本等因素都是强相关的。无论是不同的标准文件(pf_ring.ko)，目前 pf_ring驱动无法自动将内核模块标准文件部署到目标环境。以后有内核升级，也需要重新编译。

## 问题

客户环境bro占用了70%内存。软件化部署由于不确定内核和版本信息，没法预装PF_RING，而且还有可能导致bro内存超高的风险。

**PF_RING**: 一种基于Linux内核级的高效数据包捕获技术，绕过内核网络协议栈，从而减少数据的拷贝。

## 操作方法

以UOS操作系统为例：

### 1. 准备驱动源码包

3.15及以下版本请优先使用7.4.0

3.16版本：3.16由于ats容器里面预置的库文件是8.6.1的

### 2. 在lab环境搭建和线上环境一模一样的操作系统

执行make进行编译，编译完成会生成 `pf_ring.ko` 文件

### 3. 拷贝pf_ring.ko文件

线上环境可以直接操作。把pf_ring.ko文件拷贝到:

```bash
/lib/modules/<内核版本>/
```

然后执行：

```bash
depmod -a  #重新加载
```

说明：不同的操作系统可能拷贝的kernel路径不一样

### 4. 加载pf_ring内核文件，并查看是否生效

```bash
modprobe pf_ring
lsmod | grep pf_ring
```

### 5. 重启ats容器生效配置

```bash
docker restart appliance_ats_1
```

### 6. 确认pf_ring是否被调用

```bash
dcin ats
pfcount -i ens35  #ens35换成现场的p2口
```

### 7. 评伋bro线程是否满足流量要求

```bash
cat /proc/net/pf_ring/* |grep Offset
```
