# 高可用问题汇总

## DB服务器配置要求

- CPU：最少8核
- 内存：最少16G（8G测试ES起不来）
- 硬盘：
  - 必须两块盘，一块用于存储集群数据和ES、redis
  - 大小根据使用情况决定，生产环境nfs用于保存证据
  - 硬盘大小一旦决定，运行集群后无法修改，尽量大一点

## 镜像说明

- 3.10之前都可以用3.9的DB ISO，3.10及3.10之后版本使用对应版本镜像
- DB数据库镜像包（3.10之前UCSS版本用3.9的db iso build\db\3.9）
- 备注：-no-rook结尾的iso文件是不提供NFS服务的

## 注意事项

### 1. route_id的问题

- DB服务器的keepalived生成的route_id是安装的时候生成的
- 可以通过后台直接把两个地方的$router_id直接修改或者后台修改

### 2. DB高可用主要文件路径

- `/opt/skyguard/` - ES、PG等组件安装路径
- `/opt/local-path-provisioner` - 日志、事件数据等

### 3. 查看DB服务器部署日志

```bash
/var/log/deploy-cluster.log
```

### 4. 查看节点运行状态

```bash
kubectl get nodes
```

## 常见故障排查

- 检查集群状态
- 查看日志信息
- 确认网络连通性
- 检查磁盘空间
