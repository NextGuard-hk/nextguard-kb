# KB-漏扫禁用TLS1.0支持/TLS Version 1.0 Protocol Detection修复方案

## 问题描述

漏扫平台爆TLS Version 1.0 Protocol Detection，需要禁用TLS1.0。

## 原因分析

在SSL/TLS协议中发现了严重弱点，能让黑客悄悄破译Web服务器和终端用户浏览器之间传输的加密数据。弱点主要影响TLS协议1.0版及SSL所有版本，TLS 1.1/1.2未受影响。TLS是SSL的继任者，TLS 1.0相当于SSL 3.1。

## 解决方案

### 一、针对UCSS/SWG/SWG等【9443端口】，解决办法：

```bash
sed -i 's/SSLProtocol.*$/SSLProtocol all -SSLv3 -TLSv1 -TLSv1.1/' /etc/apache2/mods-enabled/ssl.conf
service apache2 restart生效
```

### 二、针对SEG服务器【25端口】，解决办法：

1、vi /etc/postfix/main.cf，新增

```
smtpd_tls_mandatory_protocols = !SSLv2, !SSLv3, !TLSv1
smtpd_tls_protocols = !SSLv2, !SSLv3, !TLSv1
```

2、重启postfix服务

```bash
/etc/init.d/postfix restart
```

### 三、针对ITP服务器【9443端口】，解决办法：

vi /opt/skyguard/itpserver/itmapi_uwsgi.ini，在[uwsgi]标示下新增下述内容：

```
ssl-enable3 = false
ssl-option = 67108864
ssl-enable-tlsv1 = false
```

```bash
service itpserver restart
```

vi /opt/skyguard/uus-api/uwsgi.ini，在[uwsgi]标示下新增同样的内容：

```
ssl-enable3 = false
ssl-option = 67108864
ssl-enable-tlsv1 = false
```

```bash
service uus-api restart
```
