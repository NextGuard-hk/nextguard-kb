# KB-子进程/bin/dash权限高于父进程

## 问题描述

用户使用的DLP服务器被通报存在安全风险，提示：子进程/bin/dash权限高于父进程，请协助分析下，该进程是什么进程，用途是什么，以及是否可以终止该进程？

云堡平台监测发现问题主机，子进程权限比父进程更高，有可能以提权的方式在系统获得更大的权限。

## 答疑

此报告反馈的是子进程高于父进程权限的异常问题，如果子进程权限比父进程高，说明有提权操作。黑客所有的攻击，都离不开提权操作，所以说怀疑有风险。比如apache漏洞，通过apache权限用户，黑客通过apache2漏洞提权到root权限进行违规操作。

但是，sudo提权也同样会有子进程高于父进程的操作，我们平台大量的业务程序都需要通过sudo进行提权管理，是正常的业务行为，并非黑客行为，不用担心。

```
-rwxr-xr-x 1 root root 105K Mar 1 2012 /bin/dash
-rwsr-xr-x 2 root root 111K May 3 2015 /usr/bin/sudo
```
