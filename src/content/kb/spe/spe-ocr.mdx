# OCR 光学字符识别

## 重点

1. arm系统不支持ocr服务
2. 3.16用的ocr的版本是fre12，这个版本目前在海光硬件上不支持
3. kv负责接受spe转发的流量，解析到图片文件就送ocr分析，然后等待ocr分析结果

我们的新版OCR，可以做一下测试，安装问题找陈明明，使用问题联系张树民

路径：`\\172.22.113.252\GuestUpload\Sec316-ocr`

## 日志

```
/opt/skyguard/ucs/var/log/OCRService.log  #OCR日志
```

## 配置文件路径

```
/opt/skyguard/ucs/etc/OCRService.xml
```

配置示例：
```xml
<abbyyParams>
  <WorkersCount>2</WorkersCount>
  <slaves_per_license>2</slaves_per_license>
  <ProcessesCount>1</ProcessesCount>
  <enable_sharedcpu>false</enable_sharedcpu>
  <MaxAcceptDurationMs>50</MaxAcceptDurationMs>
  <use_profile>true</use_profile>
  <UseFastBinarization>true</UseFastBinarization>
</abbyyParams>
```

修改OCR的工作线程，默认是2个，如果图片太多，可以考虑调大，但是会增加cpu消耗：

```xml
<WorkersCount>2</WorkersCount>  #默认2，表示2个进程，ocr分析异常，可调整为4，最大不能超过cpu核心数
<slaves_per_license>10</slaves_per_license>  #默认2，修改为10，表示起10个线程
<ProcessesCount>1</ProcessesCount>
```

线程=min(a,b*c)
- a=WorkersCount
- b=slaves_per_license
- c=ProcessesCount

下图是调整为4进程数，10线程的情况：

```
ps -ef   #进程数
ps -eLf  #线程数
```

## 超时调整

调整OCR分析超时时间：

```
/opt/skyguard/ucs/etc/DLPServer.xml
```

```xml
<OCRServer>
  <enable>true</enable>
  <accuracy>2</accuracy>
  <maxExpireTime>60</maxExpireTime>  (默认值60，单位秒)
</OCRServer>
```

修改后重启DLP服务：
```
service dlpserver restart
```

## 大量临时文件

`/opt/skyguard/ucs/tmp/ocr` 分区100%占满，查看分区文件，发现 `/opt/skyguard/ucs/tmp/ocr/cache/_SerializedImages` 有大量临时`*.tmp`文件

解决办法：

`/opt/skyguard/ucs/etc/OCRService.xml`

```xml
<backup_incomplete_image>false</backup_incomplete_image>
```

## OCRUtil工具使用

将需要识别的文件，直接传到服务器上，使用OCRUtil识别：

```bash
cd /opt/skyguard/ucs/bin
export LD_LIBRARY_PATH=/opt/skyguard/ucs/lib  # 设置环境变量
./env.sh
./OCRUtil -B /tmp/beaulity.png 3 1 1 127.0.0.1 8826 1
```

参数说明：
- 红色：1快速，2平衡，3精确
- 蓝色：OCR线程数
- 绿色：每线程通道数

乱码显示中文：Appearance里Character encoding改UTF-8

## 终端开启OCR功能

3.9以后的话需要进到SPE docker，然后改以下配置：

远程OCR服务器后台（UCSS、UCSSLite）：

```
vi /opt/skyguard/ucs/etc/DMServer.xml
```

修改 `<ucscMode>1</ucscMode>` 中 `1-->3`

| 值 | 模式 |
|----|------|
| 1 | local 本地OCR |
| 2 | remote 远端OCR |
| 3 | mixed 混合模式包含客户端的OCR |

开启后精度模式为平衡模式，不受OCR功能里限制。

如果报错：`There is not enough space available on the shmfs/tmpfs file system`

可以先执行：
```
mount -o remount,size=2G /dev/shm
```
再跑下util

## OCR支持的格式
