# 终端打印相关排查

## 生效打印拦截要点

1. 对应的应用程序需要不信任设置打印水印：内容分析后水印
2. 策略动作是终端打印
3. 有些虚拟打印不会打印拦截，是否打印拦截取决于是否是真实打印机还是虚拟打印机（如 Microsoft Print to PDF、导出为 WPS PDF 等）

## 针对图片打印不生效问题

添加参数：

```
printing-content-extraction-method=4  # 取打印机打印内容
print-xps-enabled=true  # 与打印机直接操作
```

与打印机直接联系的函数为 `StartDocPrinterW`，可用于监控此类情形。

## 实现对未保存文件的打印审计

### 针对 Windows

添加参数：

```
printing-content-extraction-method=1  # 检测打印内容，不检查文件
```

> **注意**：此参数表示检测打印内容，不检查文件，但可能对标签匹配有影响。

## 打印卡顿问题

打印卡顿通常与终端打印拦截的内容分析有关。如果出现打印卡顿，可以检查：

1. 终端配置中的打印拦截策略是否过于复杂
2. 内容分析引擎是否超时
3. 打印机驱动是否与终端存在兼容性问题
