# Network DLP

Nextguard **Network DLP** inspects data-in-motion across your network perimeter and internal segments to prevent unauthorized transmission of sensitive information.

## Supported Protocols

| Protocol | Direction | Inspection Mode |
|----------|-----------|----------------|
| HTTP / HTTPS | Outbound | Inline + SSL Decrypt |
| FTP / FTPS | Outbound | Inline |
| SMTP / POP3 / IMAP | Both | MTA Integration |
| IM (Chat protocols) | Outbound | Proxy-based |

## How It Works

1. **Traffic Mirroring or Inline**: Network DLP sits inline (bridge/transparent) or receives mirrored traffic via SPAN/TAP.
2. **Content Inspection**: The engine applies fingerprint, keyword, regex, NLP and OCR detection policies.
3. **Policy Action**: Based on matched rules, the system can **Allow**, **Block**, **Quarantine**, or **Log & Alert**.

## Deployment Modes

- **Inline (Bridge)** - Active blocking, deployed between switch and firewall
- **ICAP Integration** - Works with existing proxy (Squid, Blue Coat, etc.)
- **MTA Relay** - For email protocol DLP (SMTP)
- **Mirror/TAP** - Passive monitoring only, no blocking

## Policy Configuration

Network DLP policies are managed centrally via the **UCSS Management Platform**:

- Define **Detection Rules** (fingerprint, regex, dictionary, NLP)
- Assign **Severity Levels** (Low / Medium / High / Critical)
- Configure **Actions** per severity (log, notify, block, encrypt)
- Set **Scope** by user group, IP range, or department

## Integration with UCSS

All Network DLP events are reported to the UCSS dashboard in real-time. Administrators can:

- View incident timeline and forensic details
- Export reports for compliance audits
- Configure automated response workflows
