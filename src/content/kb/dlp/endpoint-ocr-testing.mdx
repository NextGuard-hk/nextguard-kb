# 终端OCR功能测试指南

## 概述

本指南说明如何测试终端OCR（光学字符识别）功能，即图片解析功能。

> **注意**：适用于2.0及以上版本，仅限于监控模式。

本指南旨在引导用户或SE测试OCR功能，包括通过微信/QQ发送包含敏感信息的图片进行检测。

## 配置步骤

### 步骤1：启用OCR功能

在UCSS管理控制台上开启OCR功能：

1. 登录UCSS管理控制台
2. 导航到 **系统** > **设备管理** > **ucss20**
3. 在左侧菜单找到 **功能** > **OCR**
4. 启用OCR功能开关
5. 设置最小检测字节数为 **5000**
6. 选择OCR服务类型：**本地OCR服务**

### 步骤2：修改UCSS配置文件

通过SSH登录UCSS服务器，修改配置文件：

编辑 `/opt/skyguard/ucs/etc/DMServer.xml`，将`ucscMode`修改为3：

```xml
<ucscMode>3</ucscMode>
```

> **说明**：ucscMode值含义：1=本地(local)，2=远程(remote)，3=混合(mixed)

### 步骤3：重启服务

```bash
service dlpserver restart
```

### 步骤4：测试验证

1. 重启终端机器或等待5分钟配置自动同步
2. 上传包含敏感信息的图片进行测试
3. 确保终端上开启OCR识别

## 验证OCR是否已开启

通过SSH登录UCSS后台，检查以下配置文件：

### 检查OCR服务配置

```bash
cat /opt/skyguard/ucs/etc/OCRService.xml
```

确认`enable_unlimited_license`为`true`

### 检查DLP服务配置

```bash
cat /opt/skyguard/ucs/etc/DLPServer.xml
```

确认`IsStart`为`true`

### 检查DM服务配置

```bash
cat /opt/skyguard/ucs/etc/DMServer.xml
```

确认`ucscMode`为`3`

## 常见问题

### OCR识别不准确

- 确认图片分辨率足够高
- 检查最小检测字节数设置是否合理
- 确认图片中的文字清晰可读

### OCR服务未启动

- 检查OCRService配置文件中的license设置
- 确认服务已正确重启
- 查看服务日志排查错误原因
