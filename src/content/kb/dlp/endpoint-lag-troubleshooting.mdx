# 终端卡顿问题排查

## 终端与第三方软件兼容性问题

终端和第三方软件兼容问题引起的卡顿解决办法：

### 提升驱动权限解决驱动冲突

管理员方式运行cmd执行以下命令：

1. 停保护
2. 修改Windows注册表，调整sgeptproxy服务：

```bat
reg add "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\sgeptproxy" /v "reg_dword" /d 65535 /f
```

### 标准排查步骤

1. UCSS补丁打到最新
2. Lite补丁打到最新
3. 终端用最新版本
4. 添加注册表或者定制包：
```bat
reg add "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\sgeptproxy" /v "reg_dword" /d 65535 /f
```
5. 在终端配置里关闭行为监控
6. 安全软件加白
7. 安装后在任务管理器观察SPELite和KV_server的进程ID会否占用过多资源

## 拷贝卡顿/慢

终端文件拷贝操作出现卡顿或速度异常缓慢时的排查方法：

### 可能原因

- 驱动级别监控导致文件遍历过多
- 安全软件与终端驱动冲突
- 终端内存不足

### 解决方案

1. 为相关进程添加 `file-access=lite` 属性，将驱动级监控降级为文件级监控
2. 检查并排除安全软件冲突，相互加白
3. 确认终端电脑内存是否充足

## Office/微信卡顿问题

### 问题现象

终端电脑上Office软件经常卡顿，微信发送文字也会卡顿。

### 解决方案

1. 为Office进程（winword.exe、excel.exe等）添加 `file-access=lite` 属性
2. 为微信进程添加 `file-access=lite` 属性
3. 检查终端进程属性配置是否合理
4. 确认UCSS和Lite补丁版本是否为最新

## 内存不足导致OOM

### 问题现象

终端电脑内存不够，导致OOM（Out of Memory）错误。

### 解决办法

1. 检查SPELite和KV_server进程内存占用
2. 升级终端到最新版本
3. 调整终端监控策略，减少不必要的监控项
4. 确认终端电脑硬件配置是否满足最低要求

## 相互加白路径

当终端与安全软件（如McAfee、天擎等）冲突时，需要相互加白：

- 在安全软件中将终端安装目录加入白名单
- 在终端配置中将安全软件进程加入白名单
- 常见需要加白的软件：McAfee、天擎等
